推荐目录结构

HLSL 适合把公共结构和函数放进 .hlsli，具体 shader 文件只保留入口函数和少量阶段相关逻辑。这样 D3D11、D3D12 可以共享同一批 shader 源码。

语义和 register

HLSL 使用语义连接固定管线和 shader 阶段。顶点输入语义要和 C++ input layout 对齐；阶段间自定义数据通常用 TEXCOORDn；系统值使用 SV_ 前缀。

语义用途
POSITION/NORMAL/TEXCOORD0顶点输入语义，对应 C++ vertex layout。
SV_Position顶点 shader 输出裁剪空间位置；像素 shader 输入屏幕空间位置。
SV_Target0像素 shader 第 0 个 render target 输出。
SV_DispatchThreadIDcompute shader 当前全局线程 id。
SV_GroupThreadIDcompute shader 在线程组内部的局部 id。

register 建议按资源类型分组：b 是 constant buffer，t 是 SRV，u 是 UAV，s 是 sampler。D3D12 中还可以配合 spaceN 模拟 Vulkan 的 set。

阶段结构

Vertex Shader

VertexOutput VSMain(VertexInput input) {
    float4 worldPosition = TransformPoint(...);
    output.position = mul(sceneViewProjection, worldPosition);
    return output;
}

Pixel Shader

像素 shader 应把材质采样、PBR、阴影和最终输出分成几个函数。入口函数越短，调试越容易。

Compute Shader

compute shader 要特别注意边界判断：dispatch 的线程数量经常会向上取整，多出来的线程必须 return。

代码整理规则

  • 公共 include 使用 .hlsli，具体入口文件使用 .hlsl。
  • 常量缓冲字段按 16 字节对齐组织，优先用 float4 避免 C++/HLSL 对齐不一致。
  • 入口函数命名明确：VSMain、PSMain、CSMain，阴影 pass 用 ShadowVS、ShadowPS。
  • D3D11 可用 register(b0)；D3D12 推荐加 space0、space1，方便映射 bind group。
  • 矩阵乘法顺序固定。示例使用列主序矩阵，并写成 mul(matrix, vector)。
  • 不要在光照函数里做 gamma 校正，PBR 一律在线性空间计算。

检查清单

  1. input layout 的语义、格式、slot、offset 是否和 shader 输入完全一致。
  2. register 编号是否和 D3D11/D3D12 root signature 或绑定代码一致。
  3. cbuffer 字段是否按 16 字节对齐，C++ 侧结构是否同步。
  4. shadow map 是否使用比较采样器，深度偏移是否足够小且可调。
  5. compute shader 是否检查越界线程。
  6. 所有颜色计算是否在线性空间完成，最后一步才做 tone mapping/gamma。
