球体极点与 Index 生成详细图解

重点回答：当 stack = 0 时，这一行所有顶点都位于球顶同一个空间位置，它们为什么仍能和其他顶点组成索引？最终生成的两个三角形分别发生了什么？

先给结论

stack = 0 只表示当前行 row0 是球顶行。当前格子的 a、d 来自球顶行，所以二者位置相同；b、c 来自下一行 stack = 1，位于第一条纬度环，位置不同。

因此 (a,c,b) 是有效球顶三角形；(a,d,c) 因为 a.position == d.position，面积为 0，是退化三角形。Index buffer 允许它存在，GPU 通常不会为它产生像素。

1. 为什么 stack = 0 全在一个点

球面顶点公式

v = stack / stacks
phi = v * PI
u = slice / slices
theta = u * 2 * PI
x = sin(phi) * cos(theta)
y = cos(phi)
z = sin(phi) * sin(theta)

代入 stack = 0

phi = 0
sin(phi) = 0
cos(phi) = 1
(x,y,z) = (0,1,0)

不管 slice 让 theta 变成多少，x、z 都会乘上 0，所以这一整行顶点的 position 相同。

相同 position，不等于相同顶点

它们在顶点数组中仍是不同元素，有不同 index 和 UV。以 slices = 4 为例：

IndexstackslicePositionUV
000(0,1,0)(0.00,0)
101(0,1,0)(0.25,0)
202(0,1,0)(0.50,0)
303(0,1,0)(0.75,0)
404(0,1,0)(1.00,0)

Index 是顶点数组下标，不是空间坐标。不同 index 可以保存相同 position。复制极点顶点可以保留不同 UV，并让所有 stack 使用同一套索引公式。

2. 第一格到底取到了谁

使用 slices = 4，选择 stack = 0、slice = 0。每行有 slices + 1 = 5 个顶点。

3. stack = 0、slice = 0 的计算

计算行宽。

rowStride = slices + 1 = 5。

得到极点行。

row0 = 0 * 5 = 0，这一行包含 index 0、1、2、3、4。

得到下一纬度环。

row1 = (0 + 1) * 5 = 5，这一行包含 index 5、6、7、8、9。

取得四个角。

a=0、b=5、c=6、d=1。只有 a、d 在球顶重合；b、c 位于下一纬度环。

写入六个 index。

{a,c,b,a,d,c} 变成 {0,6,5, 0,1,6}。

有效：(a,c,b) = (0,6,5)

a 在球顶，b、c 在第一条纬度环，三个空间位置不同，因此三角形有面积。

退化：(a,d,c) = (0,1,6)

a 和 d 的 index 不同，但 position 相同。空间中只剩两个不同点，三角形被压成一条线。

用面积公式验证

area = 0.5 * length(cross(P1 - P0, P2 - P0))
Pd - Pa = (0,0,0)
cross((0,0,0), Pc - Pa) = (0,0,0)
area = 0

Index buffer 只负责每三个 index 组装一个三角形，不检查顶点位置是否重复。退化三角形可以提交给 GPU，只是光栅化阶段没有覆盖面积，通常不会产生 fragment。

4. 整个球顶怎样形成

slice 继续递增时，每个格子生成一个有效顶盖三角形和一个退化三角形。所有有效三角形共享同一个球顶空间位置，组成三角扇。

5. 其他 stack 的情况

stack = 0

a、d 在球顶重合。(a,c,b) 有效，(a,d,c) 退化。

中间 stack

上下两行都是纬度环。a、b、c、d 通常是四个不同位置，两个三角形都有效。

stack = stacks - 1

b、c 在球底重合。(a,c,b) 退化，(a,d,c) 有效。

规则网格写法

可以正常渲染。

优点是公式统一、代码简单、UV 接缝好处理。代价是顶部和底部各产生 slices 个退化三角形，并保存重复极点顶点。

无退化三角形写法

顶部只保存一个极点，底部只保存一个极点；顶盖和底盖单独生成三角扇，中间纬度环按四边形拆分。这样顶点和 index 更少，但代码会分为顶盖、中间、底盖三部分。

一句话记忆

球顶行是“多个不同 index 的重复顶点”，当前格子的另外两个顶点来自下一纬度环。所以一个三角形有效；另一个因为两个极点 position 重合而退化。
