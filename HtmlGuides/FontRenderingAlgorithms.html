Bitmap / SDF / MSDF 字体渲染算法

字体渲染最终仍然是渲染一批 Quad：每个字符生成 4 个顶点和 6 个索引，通过 UV 采样字体图集。Bitmap、SDF、MSDF 的区别在于字体图集里存的是什么，以及 fragment shader 如何把采样值变成最终 alpha。

共同流程

无论 Bitmap、SDF 还是 MSDF，核心流程都一样：把字体文件中的矢量字形转换成纹理图集，然后每个字符画一个 Quad。

1. 字体文件 读取 .ttf / .otf，得到 glyph outline 和字体度量。
2. 字形生成 把字符转换成 bitmap、SDF 或 MSDF 像素数据。
3. 图集打包 把多个字符放入同一张 atlas，记录每个字符的 UV。
4. 文本排版 根据 advance、bearing、kerning 计算每个字符的位置。
5. Quad Mesh 每个字符生成一个矩形，两组三角形，顶点带 position、uv、color。
6. Shader 采样 fragment shader 根据采样值计算 alpha，再和文字颜色混合。 
Font file (.ttf/.otf)
  -> glyph rasterizer / distance field generator
  -> font atlas texture
  -> glyph metrics table
  -> text layout
  -> vertex/index buffer
  -> draw triangles

字体渲染不是把整个字符串画成一张图片再贴上去。实时 UI 更常见的做法是：图集提前生成，每次字符串变化时只重新生成字符 quad。

字体数据

字体渲染至少需要两类数据：字体图集和每个 glyph 的度量信息。

Glyph 数据结构

struct Glyph
{
    uint32_t codepoint;      // Unicode 字符，例如 'A' 或中文字符码点

    float advanceX;          // 画完这个字符后，penX 往右移动多少
    float bearingX;          // 字符左边界相对 penX 的偏移
    float bearingY;          // 字符上边界相对 baseline 的偏移
    float width;             // 字形 Quad 宽度
    float height;            // 字形 Quad 高度

    float u0, v0;            // atlas 左上 UV
    float u1, v1;            // atlas 右下 UV
};

顶点数据结构

struct TextVertex
{
    float2 position;         // UI 中是像素坐标；3D 中可以是局部平面坐标
    float2 uv;               // 指向字体 atlas
    float4 color;            // 文字颜色
};

关键概念

概念 含义 作用
Baseline 文字排版的基线，大部分拉丁字符站在这条线上。 决定字符上下位置。
Advance 绘制当前字符后 pen 前进距离。 决定下一个字符起点。
Bearing glyph 实际图像相对 pen 的偏移。 决定字符 quad 的左上角位置。
Kerning 特定字符对之间的额外间距调整。 让 AV、To 等组合看起来更自然。
Atlas Padding 每个字符周围留空。 避免采样串色；SDF/MSDF 还需要给距离场留范围。

Bitmap 字体

Bitmap 字体是最直接的方法：图集里存每个字符的灰度或 alpha。shader 采样 alpha，乘上文字颜色输出。

最简单 小字号清晰 缩放会糊

图集存储内容

atlas.r 或 atlas.a =
    0.0 表示完全透明
    1.0 表示字体笔画内部
    0..1 表示抗锯齿边缘

Bitmap Shader

// GLSL/HLSL 风格伪代码
float glyphAlpha = Sample(fontAtlas, input.uv).r;
float4 color = input.color;
color.a *= glyphAlpha;
return color;

优点

  • 实现最简单，几乎不需要特殊 shader。
  • 小字号下效果非常直接，适合固定尺寸 UI。
  • 生成速度快，图集调试容易。

缺点

  • 放大会模糊，缩小会闪烁或丢细节。
  • 每个字号最好生成一套图集，否则清晰度不好。
  • 描边、阴影、发光需要额外图集或额外处理。

适合场景

固定尺寸 UI、小字号文本、调试文字、性能要求极低的简单工具界面。

SDF 字体

SDF = Signed Distance Field，符号距离场。图集里不再直接存“是不是字”，而是存“当前像素到字形边缘的距离”。

缩放清晰 描边简单 尖角容易圆

图集存储内容

distance = 0.5  表示字形边缘
distance > 0.5  表示字形内部
distance < 0.5  表示字形外部

实际距离会被映射到 0..1，常用 0.5 作为边界。shader 用 smoothstep 根据距离重新生成边缘 alpha。

SDF Shader

float d = Sample(fontAtlas, input.uv).r;

// fwidth 会根据屏幕像素变化估算过渡宽度，让边缘抗锯齿更稳定。
float width = fwidth(d);
float alpha = smoothstep(0.5 - width, 0.5 + width, d);

float4 color = input.color;
color.a *= alpha;
return color;

固定平滑宽度版本

float d = Sample(fontAtlas, input.uv).r;
float smoothing = 0.02;
float alpha = smoothstep(0.5 - smoothing, 0.5 + smoothing, d);

使用 fwidth 的版本通常更适合不同缩放比例；固定 smoothing 更容易理解，但缩放变化大时可能过糊或过硬。

优点

  • 同一张图集可以支持多个字号和缩放。
  • 描边、阴影、外发光可以通过距离范围直接做。
  • 图集只需要单通道，内存比 MSDF 少。

缺点

  • 尖角、细小内角容易变圆。
  • 极小字号可能不如专门生成的 bitmap 清晰。
  • 距离范围、padding、图集分辨率需要调好。

适合场景

游戏 UI、可缩放文字、3D 世界文字、简单描边和阴影文字。

MSDF 字体

MSDF = Multi-channel Signed Distance Field，多通道符号距离场。它用 RGB 三个通道分别编码不同边缘的距离信息，从而更好保留尖角。

尖角清晰 高质量缩放 生成更复杂

图集存储内容

atlas.r = edge group 0 distance
atlas.g = edge group 1 distance
atlas.b = edge group 2 distance

边界仍然大约在 0.5
shader 取 RGB 的 median 作为最终距离

Median 函数

float Median(float r, float g, float b)
{
    return max(min(r, g), min(max(r, g), b));
}

MSDF Shader

float3 sampleColor = Sample(fontAtlas, input.uv).rgb;
float d = Median(sampleColor.r, sampleColor.g, sampleColor.b);

float width = fwidth(d);
float alpha = smoothstep(0.5 - width, 0.5 + width, d);

float4 color = input.color;
color.a *= alpha;
return color;

MTSDF

MTSDF 是 MSDF 的增强变体，通常 RGB 存 MSDF，A 通道存普通 SDF。这样可以同时保留尖角，并更方便做外描边、阴影和发光。

float3 msdf = Sample(fontAtlas, uv).rgb;
float sdf = Sample(fontAtlas, uv).a;

float sharpDistance = Median(msdf.r, msdf.g, msdf.b);
float softDistance = sdf;

优点

  • 尖角和硬边比 SDF 更清晰。
  • 适合 UI 大标题、3D 字体、可缩放编辑器文字。
  • 同一套 atlas 可以覆盖更大的显示尺寸范围。

缺点

  • 生成算法更复杂，通常需要专门工具。
  • 纹理至少 RGB 三通道，内存比单通道 SDF 高。
  • shader 要正确取 median；mipmap、采样、padding 不当会出彩边。

适合场景

高质量 UI、编辑器 UI、标题文字、3D 文字、需要大范围缩放且要保留锐利边角的项目。

三种对比

算法 图集存储 Shader 核心 优点 缺点 推荐使用
Bitmap 字形 alpha。 alpha = texture.r 最简单，小字号清楚，调试容易。 缩放差，不同字号要多套图集。 调试 UI、固定字号、小工具。
SDF 单通道距离场。 smoothstep(edge, distance) 缩放好，描边/阴影简单。 尖角容易圆，极小字号不一定好。 游戏 UI、3D 名字牌、普通可缩放文字。
MSDF RGB 多通道距离场。 distance = median(rgb) 尖角清晰，缩放质量高。 生成复杂，内存更高，采样更敏感。 编辑器 UI、高质量 UI、3D 文字、大标题。

自研引擎建议路线：先实现 Bitmap，把排版、atlas、quad、batch、blend 跑通；然后升级 SDF；最后接 MSDF/MTSDF 做高质量版本。

排版与 Quad

渲染字符串时，CPU 根据 glyph metrics 生成每个字符的四个顶点。

void AppendGlyphQuad(
    const Glyph& glyph,
    float penX,
    float baselineY,
    Color color,
    std::vector<TextVertex>& vertices,
    std::vector<uint32_t>& indices)
{
    float x0 = penX + glyph.bearingX;
    float y0 = baselineY - glyph.bearingY;
    float x1 = x0 + glyph.width;
    float y1 = y0 + glyph.height;

    uint32_t base = (uint32_t)vertices.size();

    vertices.push_back({ { x0, y0 }, { glyph.u0, glyph.v0 }, color });
    vertices.push_back({ { x1, y0 }, { glyph.u1, glyph.v0 }, color });
    vertices.push_back({ { x1, y1 }, { glyph.u1, glyph.v1 }, color });
    vertices.push_back({ { x0, y1 }, { glyph.u0, glyph.v1 }, color });

    indices.push_back(base + 0);
    indices.push_back(base + 1);
    indices.push_back(base + 2);
    indices.push_back(base + 0);
    indices.push_back(base + 2);
    indices.push_back(base + 3);
}

字符串布局

float penX = startX;
float baselineY = startY;

for each codepoint in text:
    if codepoint == '\n':
        penX = startX;
        baselineY += lineHeight;
        continue;

    Glyph glyph = font.GetGlyph(codepoint);

    if previousCodepoint exists:
        penX += font.GetKerning(previousCodepoint, codepoint);

    AppendGlyphQuad(glyph, penX, baselineY, color, vertices, indices);
    penX += glyph.advanceX;

屏幕坐标到 NDC

float2 PixelToNdc(float2 p, float2 screenSize)
{
    float x = p.x / screenSize.x * 2.0 - 1.0;
    float y = 1.0 - p.y / screenSize.y * 2.0;
    return float2(x, y);
}

UI 与 3D 文字

类型 顶点坐标 矩阵 常见处理
UI 文字 屏幕像素坐标或 UI 局部坐标。 屏幕正交投影，或 shader 中根据 screenSize 转 NDC。 关闭深度测试，开启 alpha blend，按 UI 层级排序。
3D 文字 世界空间、模型局部空间或 billboard 平面。 正常走 Model/View/Projection。 可开启深度测试；名字牌常使用 billboard 朝向相机。
屏幕贴附 3D 标签 先把 3D 点投影到屏幕，再用 UI 方式画文字。 3D position -> clip -> screen，然后 UI 绘制。 适合血条、名字、交互提示。

Billboard 文字

// center 是文字牌在世界空间的位置
// right/up 来自相机方向
worldPos = center + cameraRight * localPos.x + cameraUp * localPos.y;

描边、阴影、发光

SDF 描边

float d = Sample(fontAtlas, uv).r;
float aa = fwidth(d);

float fill = smoothstep(0.5 - aa, 0.5 + aa, d);

float outlineEdge = 0.5 - outlineWidth;
float outline = smoothstep(outlineEdge - aa, outlineEdge + aa, d);

float outlineOnly = outline - fill;
float4 result =
    fill * fillColor +
    outlineOnly * outlineColor;

SDF 阴影

float fillDistance = Sample(fontAtlas, uv).r;
float shadowDistance = Sample(fontAtlas, uv + shadowOffset).r;

float aa = fwidth(fillDistance);
float fillAlpha = smoothstep(0.5 - aa, 0.5 + aa, fillDistance);
float shadowAlpha = smoothstep(0.5 - aa, 0.5 + aa, shadowDistance);

float4 color = shadowColor * shadowAlpha;
color = lerp(color, fillColor, fillAlpha);

MSDF 描边注意

MSDF 的 RGB median 非常适合字形主体锐利边缘；如果要做大范围外描边和发光，MTSDF 的 alpha 通道 SDF 更稳定。

引擎实现建议

模块拆分

模块 职责 关键数据
FontLoader 读取 .ttf/.otf，获得 glyph outline、metrics、kerning。 face、size、ascender、descender、lineHeight。
FontAtlasBuilder 生成 Bitmap/SDF/MSDF 图集并打包。 atlas texture、glyph UV、padding、distance range。
TextLayout 处理字符串、换行、对齐、kerning、裁剪。 glyph runs、line boxes、text bounds。
TextMeshBuilder 把排版结果转换成 vertex/index buffer。 TextVertex、indices、draw ranges。
TextRenderer 按 atlas、材质、clip rect 分批绘制。 pipeline、sampler、blend、uniform。

推荐实现路线

  1. 先支持 ASCII Bitmap 字体，把 Quad、UV、Blend、UI 坐标跑通。
  2. 加入 UTF-8 解码，支持 Unicode codepoint。
  3. 接入 FreeType 或 stb_truetype 生成 glyph metrics。
  4. 支持动态 atlas，缺字时运行时加入 glyph。
  5. 升级 SDF shader，加入描边和阴影。
  6. 最后支持 MSDF/MTSDF，处理高质量缩放和复杂效果。

中文字体字符数量很大，不适合一次性把所有汉字都塞进图集。实际引擎常用动态字体图集：用到哪个字就生成哪个字，图集满了再换页或重建。

调试清单

  • 先显示字体 atlas，确认字符没有互相覆盖，padding 足够。
  • 用纯白文字显示 Bitmap/SDF/MSDF 主体，先不要开描边和阴影。
  • 检查 UV 是否上下翻转，尤其是图片加载和图集坐标原点。
  • 检查 alpha blend：常用 srcAlpha, oneMinusSrcAlpha。
  • 检查 sRGB：字体 atlas 通常是数据纹理，不应按 sRGB 解码。
  • SDF/MSDF 缩放异常时，先检查 padding、distance range、mipmap。
  • MSDF 出现彩边时，检查 median shader、线性采样、图集边缘 padding。
  • 文字位置跳动时，检查 bearing、advance、baseline、kerning 和像素对齐。
