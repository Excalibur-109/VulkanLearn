下载格式选择

格式 优先级 原因 注意点
.glb 最高 单文件包含 mesh、material、texture 引用或嵌入数据，最适合引擎 demo。 文件可能较大；调试时不如外部纹理直观。
.gltf + textures/ 最高 结构清楚，贴图文件外置，便于你检查 BaseColor、Normal、Roughness 等。 移动文件夹时要保持相对路径。
.fbx 中 游戏美术常见格式，动画和骨骼支持较多。 不同导出器差异大，材质和贴图经常需要在 Blender 里重新整理。
.obj + .mtl 中低 简单、通用，适合静态 mesh。 材质能力弱，很多只有 diffuse，不适合完整 PBR 测试。
.blend 整理用 适合在 Blender 中修模型、合并材质、重新导出 glTF。 不建议作为引擎直接加载格式。

贴图命名检查

下载后先检查文件夹里有没有下面这些贴图。你的引擎先支持这些就能覆盖大部分 PBR 资产。

BaseColor / Albedo / Diffuse      // 基础颜色，通常是 sRGB
Normal / NormalGL / NormalDX      // 法线贴图，注意 OpenGL 和 DirectX 的 Y 方向
Roughness                         // 粗糙度，线性纹理
Metallic / Metalness              // 金属度，线性纹理
AO / Occlusion                    // 环境遮蔽，线性纹理
Emissive / Emission               // 自发光，可选
Height / Displacement             // 高度或位移，可选

glTF 的 metallic-roughness 工作流通常把 Metallic 和 Roughness 合并在同一张纹理中：B 通道是 metallic，G 通道是 roughness。导入器要按 glTF 规则解包，不要按文件名硬猜。

导入自研引擎的建议流程

  1. 先下载 glTF/GLB 模型，避免一开始被 FBX 材质导入问题干扰。
  2. 用 Blender 打开模型，确认 mesh、材质、贴图和 UV 正常。
  3. 导出一份干净的 .gltf + textures/，用于观察真实文件结构。
  4. 引擎侧先支持静态 mesh、单材质、多贴图、sampler、mipmap、sRGB/linear 区分。
  5. 再支持多 mesh、多 material、法线贴图切线空间、透明材质、双面材质。
  6. 最后支持骨骼动画、实例化、材质参数编辑、资源热重载。

许可证提醒

下载素材时不要只看“free”。免费可能只是免费下载，不一定允许商用，也不一定允许重新分发源文件。尤其是你后面如果做自己的引擎 demo、教学项目、游戏发布，需要保留每个资产的来源和 license。

许可类型 通常含义 你要做什么
CC0 / Public Domain 最宽松，通常可商用、可修改、无需署名。 仍建议记录来源 URL，方便以后追溯。
CC-BY 通常可用，但必须署名。 记录作者、标题、链接和 license。
CC-BY-SA 通常要求相同方式共享衍生作品。 商业项目慎用，先确认是否符合你的发布方式。
Royalty-Free 通常可集成到作品里，但不能单独转售模型本体。 看具体站点条款，尤其是是否允许重新分发源资产。
