卡通渲染全流程逻辑思路

这份文档只讲渲染流程、算法核心和工程组织，不绑定任何图形 API。你可以把它理解成自研引擎里的 Toon Render Design Doc：先决定美术风格，再定义输入数据，然后逐层实现表面颜色、卡通光照、阴影、描边、特殊材质和后处理。

目录 目标定义 总流程 输入数据 资产准备 表面颜色 卡通光照 阴影处理 描边系统 脸部与头发 特殊材质 后处理 稳定性 引擎模块 调试顺序 常见问题

目标定义

卡通渲染不是简单地把光照做成几段。它的核心是“把三维结果约束成可控的二维美术表达”。写实 PBR 追求物理合理，卡通渲染追求形状清晰、色块稳定、明暗好看、轮廓有设计感。

目标 算法含义 工程含义
大色块 把连续光照量化为亮面、半影、暗面几段。 需要 ramp、阈值、平滑宽度和材质参数。
轮廓清晰 通过模型外扩、法线/深度边缘或几何边缘产生线条。 需要独立描边 pass 或屏幕空间边缘检测。
颜色可控 阴影颜色不只是乘黑，而是由色阶或 ramp 决定。 材质要有 base、shadow、rim、outline 等参数。
角色好看 脸部、头发、眼睛不完全遵守真实光照。 需要角色专用规则，例如脸部 SDF 阴影和头发高光。
运动稳定 色块、线条、阴影在相机移动时不闪烁。 需要抗锯齿、阈值平滑、法线修正和 LOD 策略。

总流程

不考虑图形 API 时，卡通渲染可以拆成下面这些逻辑阶段。每个阶段都应该能单独调试。

1. 资产输入 mesh、UV、normal、tangent、vertex color、材质贴图、角色控制图。
2. 表面建模 得到 base color、shadow color、normal、roughness、style mask。
3. 主光量化 计算 N dot L，把连续光照变成卡通色阶或 ramp。
4. 阴影修饰 把 shadow map、AO、角色规则融合到暗面判断。
5. 线条生成 外轮廓、内部结构线、材质分界线、屏幕空间边缘。
6. 后处理 颜色分级、边缘抗锯齿、轻量 bloom、最终风格统一。 
FinalColor =
    ToonSurfaceColor
  + StylizedDirectLight
  + StylizedSpecular
  + RimLight
  + Emission
  - OutlineAndInkContribution
  + PostProcessAdjustments

卡通渲染的关键不是公式复杂，而是每个步骤都给美术留参数：亮暗阈值、过渡宽度、暗部颜色、边线宽度、边线颜色、脸部阴影方向、头发高光形状。

输入数据

要做稳定的卡通效果，模型数据比 shader 公式更重要。许多“二次元味道”来自被美术修过的法线、顶点色和专用贴图。

数据 用途 常见细节
Position 几何位置，用于轮廓外扩、深度、阴影和距离衰减。 角色模型的比例会直接影响线宽和阴影观感。
Normal 控制明暗分界，是卡通渲染最重要的输入之一。 常常需要编辑法线，让脸、衣服、头发的明暗块更干净。
Tangent 支持法线贴图、头发方向、高光方向。 导入时要保证 tangent space 和法线贴图方向一致。
UV 采样 base、ramp、mask、SDF、detail 等贴图。 角色经常会有多套 UV，例如一套颜色，一套脸部 SDF。
Vertex Color 轻量控制区域参数，例如线宽、暗部权重、AO、材质 ID。 适合存不需要高清贴图的风格控制信息。
Base Color 主颜色，决定角色和物体的固有色。 通常颜色更干净，细节少于写实贴图。
Ramp Texture 把光照强度映射成卡通色带。 可以一维 ramp，也可以二维 ramp：横轴光照，纵轴材质区域。
Style Mask 控制不同部位使用不同阈值、线宽、阴影颜色。 常把多个 mask 打包到 RGBA 通道。
Face SDF 脸部专用阴影，避免鼻子、嘴、脸颊被真实法线切碎。 常用于二次元角色，根据光照方向采样 SDF 决定脸部暗面。

资产准备

1. 法线编辑

卡通角色经常不直接使用几何真实法线。真实法线会让脸部、衣服褶皱和头发出现碎光斑，而卡通渲染需要清晰大块明暗。

  • 脸部法线可以向前或向头部中心平滑，减少鼻梁和嘴部造成的破碎明暗。
  • 头发法线可以按发束方向或头部球面重建，让高光形成可控带状。
  • 衣服硬折线处可以保留硬法线，让结构线更明确。
  • 外轮廓需要单独考虑，外扩描边常依赖平滑法线或自定义 outline normal。

2. 贴图拆分

建议从一开始就把“颜色”和“风格控制”分开。BaseColor 负责颜色，Mask 负责算法选择。

BaseColor.rgb      // 主颜色
Mask.r             // 暗部强度或 shadow threshold offset
Mask.g             // 高光强度或 specular mask
Mask.b             // 边缘光强度或 rim mask
Mask.a             // 描边宽度或材质区域 ID
VertexColor.r      // 局部线宽
VertexColor.g      // 局部阴影偏移
VertexColor.b      // 局部 AO
VertexColor.a      // 预留

3. 材质分类

卡通渲染中不同材质不是只换贴图，往往要换光照规则。

材质 推荐规则 关键参数
皮肤 柔和阴影，低强度高光，脸部可走 SDF 阴影。 skinShadowColor、faceShadowWidth、softness。
头发 方向性高光，额外 rim，阴影边界可以更硬。 hairSpecShift、highlightWidth、rimStrength。
布料 稳定明暗块，轻微 AO，少量或无高光。 shadowThreshold、shadowColor、outlineWidth。
金属 非真实反射，可使用 MatCap 或色带高光。 matcapStrength、specBand、edgeTint。
眼睛 通常不走普通光照，使用手绘贴图、固定高光、透明层。 eyeHighlight、irisColor、emission。

表面颜色

表面颜色阶段只回答一个问题：这个像素在没有光照风格化前是什么材质。卡通渲染不一定需要完整 PBR，但仍需要清晰的材质输入。

SurfaceData BuildSurface(Input input)
{
    SurfaceData s;
    s.baseColor       = SampleBaseColor(input.uv) * material.baseColorTint;
    s.normal          = Normalize(ApplyNormalMap(input.normal, input.tangent, input.uv));
    s.shadowColor     = material.shadowColor;
    s.rampIndex       = material.rampIndex;
    s.shadowThreshold = material.shadowThreshold + SampleMask(input.uv).r;
    s.specularMask    = SampleMask(input.uv).g;
    s.rimMask         = SampleMask(input.uv).b;
    s.outlineWidth    = material.outlineWidth * (1.0 + SampleMask(input.uv).a);
    return s;
}

如果你直接把写实 PBR 的 albedo、roughness、metallic 塞进卡通公式，效果可能会脏。卡通资产通常需要更干净的 base color、更少的噪声和更明确的材质区域。

卡通光照

1. 基础 N dot L

最基础的光照强度来自法线和主光方向的夹角。

N = normalize(surface.normal)
L = normalize(mainLight.direction)
rawLight = dot(N, L)                // 范围通常是 -1 到 1
halfLambert = rawLight * 0.5 + 0.5  // 转成 0 到 1，更适合卡通色阶

2. 阶梯量化

把连续光照变成固定色阶。最简单是二分：亮面和暗面。

light01 = saturate(dot(N, L) * 0.5 + 0.5)
threshold = material.shadowThreshold
softness = material.shadowSoftness

// softness 越小，边界越硬；越大，过渡越柔。
toonLight = smoothstep(threshold - softness, threshold + softness, light01)

litColor    = baseColor * material.lightTint
shadowColor = baseColor * material.shadowTint
diffuse     = lerp(shadowColor, litColor, toonLight)

3. 多段色阶

二分会很干净，但有时太硬。三段或四段更适合立体感强的风格。

if light01 < shadowThreshold:
    band = 0        // 暗面
else if light01 < midThreshold:
    band = 1        // 半影
else:
    band = 2        // 亮面

diffuse = baseColor * bandColor[band]

4. Ramp Texture

Ramp 是卡通渲染中非常实用的工具。它把 light01 映射到一条颜色带，艺术家可以直接画出亮面、暗面和过渡颜色。

u = saturate(light01 + material.rampOffset)
v = material.rampIndex
rampColor = SampleRampTexture(float2(u, v))
diffuse = baseColor * rampColor
方案 优点 缺点
硬阈值 便宜、清楚、容易理解。 容易锯齿和闪烁。
Smoothstep 阈值 边界更稳，可控制软硬。 过软会失去卡通色块感。
多段 band 层次更强，适合角色和场景。 参数多，需要美术调试。
Ramp Texture 美术可控，风格统一。 需要管理 ramp 资源和采样规则。

5. 高光

卡通高光通常不是连续 Blinn-Phong，而是经过阈值化的高光块。

V = normalize(cameraPosition - worldPosition)
H = normalize(L + V)
specRaw = pow(saturate(dot(N, H)), material.specPower)

specBand = smoothstep(
    material.specThreshold - material.specSoftness,
    material.specThreshold + material.specSoftness,
    specRaw
)

specular = material.specColor * specBand * surface.specularMask

6. 边缘光

Rim Light 用视线方向制造角色轮廓亮边，常用于把角色从背景里分离出来。

rimRaw = 1.0 - saturate(dot(N, V))
rimShape = pow(rimRaw, material.rimPower)
rimBand = smoothstep(material.rimThreshold, material.rimThreshold + material.rimSoftness, rimShape)
rim = material.rimColor * rimBand * surface.rimMask

阴影处理

卡通阴影分两类：一类是光照色阶产生的暗面，一类是真实遮挡产生的投影。两者需要融合，但不能简单相乘到发黑。

1. 自身明暗和投影融合

toonLight = QuantizeLight(dot(N, L))
shadowVisibility = SampleShadow(worldPosition)  // 1 表示无遮挡，0 表示被阴影挡住

// 投影不要直接乘黑，而是把它推向暗面。
shadowInfluence = 1.0 - shadowVisibility
finalLightFactor = min(toonLight, 1.0 - shadowInfluence * material.castShadowStrength)

diffuse = lerp(shadowColor, litColor, finalLightFactor)

2. 阴影颜色

暗部颜色应该可控，通常带一点色相偏移，而不是简单 baseColor * 0.3。

方式 做法 适合情况
乘色 shadowColor = baseColor * shadowTint 快速实现，适合原型。
贴图指定 shadowColor = SampleShadowColorMap(uv) 角色皮肤、衣服需要精细暗部颜色。
Ramp 指定 shadowColor = baseColor * SampleRamp(light01) 希望全项目风格统一。

3. AO 的使用

AO 可以增强接触感，但不能破坏卡通大色块。建议只影响暗面强度或局部接触阴影，不要在亮面制造太多脏细节。

ao = SampleAO(uv)
aoShadow = lerp(1.0, ao, material.aoStrength)
shadowColor *= aoShadow

常见错误：把 shadow map、AO、light attenuation、PBR diffuse 全部相乘，结果会变成脏黑的写实阴影。卡通渲染更适合“选择暗部颜色”，不是“不断乘黑”。

描边系统

描边是卡通渲染的核心组成部分。通常不会只用一种方法，而是组合外轮廓、内部边缘和手绘结构线。

1. 外扩描边

把模型沿法线或自定义 outline normal 外扩，渲染背面，得到稳定外轮廓。

outlinePosition = position + outlineNormal * outlineWidth
outlineColor = material.outlineColor

// 线宽可以受距离影响，避免远处线太粗或近处线太细。
distanceScale = ComputeDistanceScale(cameraDistance)
finalWidth = baseWidth * vertexWidthMask * distanceScale
优点 缺点 适合
线条稳定，角色外轮廓好看，艺术可控。 内轮廓不明显，尖角和开口模型可能出问题。 角色、道具、低模风格。

2. 屏幕空间边缘检测

根据深度、法线、材质 ID 的突变检测边缘。适合补内部结构线。

centerDepth = Depth(x, y)
centerNormal = Normal(x, y)
centerId = MaterialId(x, y)

edge = 0
for each neighbor around pixel:
    depthDiff = abs(centerDepth - Depth(neighbor))
    normalDiff = 1.0 - dot(centerNormal, Normal(neighbor))
    idDiff = centerId != MaterialId(neighbor)

    edge += depthDiff > depthThreshold
    edge += normalDiff > normalThreshold
    edge += idDiff

outlineMask = saturate(edge)
边缘来源 能检测什么 风险
Depth 物体外边界、遮挡边界。 远处深度精度、斜面边缘容易不稳定。
Normal 模型内部折线、硬边、结构线。 法线贴图太细会产生噪声线。
Material ID 不同材质区域分界。 需要额外 ID 数据，透明物体要单独处理。

3. 手绘结构线

最可控的内部线条通常来自贴图或模型几何，而不是自动算法。

  • 衣服缝线、眼线、嘴线、发束线可以画进 base color 或单独 ink mask。
  • 结构线可以用 mesh strip 做，这样线条随动画稳定。
  • 复杂角色常用外扩描边负责外轮廓，贴图线负责脸和衣服细节，屏幕空间线负责环境物体。

4. 线宽控制

finalOutlineWidth =
    material.baseOutlineWidth
  * vertexColor.r
  * distanceCompensation
  * perspectiveCompensation
  * styleGlobalScale

外扩描边的线宽最好有全局缩放参数。这样你可以针对不同分辨率、镜头距离、角色大小统一调线宽。

脸部与头发

1. 脸部为什么要特殊处理

真实法线光照会让脸部出现鼻子、嘴唇、眼窝带来的碎阴影。卡通角色脸部通常追求干净、稳定、可控，所以常用 SDF 或特殊 mask 来决定脸部阴影。

2. 脸部 SDF 阴影

Face SDF 的基本思路：在一张贴图中编码脸部左右方向的阴影边界，根据光照方向决定采样和阈值。

faceRight = normalize(characterRight)
faceForward = normalize(characterForward)
lightDir = normalize(mainLight.direction)

rightAmount = dot(lightDir, faceRight)
frontAmount = dot(lightDir, faceForward)

uv = faceUv
if rightAmount < 0:
    uv.x = 1.0 - uv.x

sdf = SampleFaceSDF(uv)
threshold = RemapFrontLight(frontAmount)
faceShadow = step(threshold, sdf)

faceColor = lerp(faceShadowColor, faceLitColor, faceShadow)

这样脸部阴影更像画出来的形状，而不是由鼻子模型真实投影出来的形状。

3. 头发高光

头发高光可以用两类方法：基于方向的各向异性高光，或者直接使用头发高光 mask。

T = normalize(hairTangent)
H = normalize(L + V)
strandSpec = pow(saturate(1.0 - abs(dot(T, H))), hairSpecPower)
hairBand = smoothstep(hairThreshold, hairThreshold + hairSoftness, strandSpec)
hairSpecular = hairHighlightColor * hairBand * SampleHairSpecMask(uv)

如果你先做引擎原型，建议头发高光先用贴图 mask 控制，等基础渲染稳定后再做复杂各向异性。

特殊材质

类型 算法核心 实现要点
眼睛 弱化真实光照，使用手绘虹膜、固定高光和轻微视差。 眼白、虹膜、高光可以拆层；高光可以固定在屏幕或角色局部方向。
金属 用 MatCap、色带反射或阈值高光代替真实环境反射。 保持高对比，但不要出现脏噪声反射。
玻璃 透明、边缘亮线、简化反射。 卡通玻璃常用 rim 和固定高光，不追求完整折射。
能量特效 Emission、Fresnel、噪声流动、软边缘。 使用独立颜色和 bloom 控制，避免污染普通材质曝光。
水面 手绘法线、边缘泡沫、深浅颜色分层。 卡通水比写实水更依赖颜色分区和泡沫线。

MatCap 思路

MatCap 用视空间法线当作二维坐标去查一张“材质球贴图”，可以快速得到卡通金属、塑料或特殊材质。

viewNormal = TransformNormalToViewSpace(N)
matcapUv = viewNormal.xy * 0.5 + 0.5
matcapColor = SampleMatCap(matcapUv)
finalColor = lerp(toonColor, matcapColor * baseColor, material.matcapStrength)

后处理

卡通后处理的目标是统一画面，而不是把画面做得更写实。推荐先少做，再逐步加。

后处理 用途 注意点
Color Grading 统一全局色调，让角色和场景更像同一个作品。 不要破坏材质 ramp 的明暗关系。
FXAA / TAA 减少描边、色阶边界、几何边缘锯齿。 TAA 可能让线条变糊，需要调锐化和历史权重。
Bloom 用于眼睛高光、魔法、能量、发光道具。 强度要低，避免整个画面泛白。
Depth Fog 场景层次和远景简化。 风格化雾色通常比真实雾更干净。
Vignette 轻微聚焦视线。 不要变成明显黑边，除非美术风格需要。

稳定性

卡通渲染最容易出问题的地方是闪烁：明暗边界闪、描边闪、阴影闪、法线贴图造成细碎边缘。稳定性需要从算法和资产两边处理。

问题 原因 处理方式
明暗边界闪烁 硬阈值对法线和相机变化非常敏感。 用 smoothstep 加一点过渡宽度；远处降低法线细节。
描边粗细跳动 线宽按世界空间固定，透视和距离变化明显。 引入屏幕空间线宽目标，按距离补偿。
内部线噪声 屏幕空间 normal edge 检测采到了法线贴图高频。 边缘检测使用几何法线，或提高 normal threshold。
阴影边缘抖动 阴影采样精度、级联切换、bias 不稳定。 卡通阴影可增加边界平滑和固定级联过渡。
远处色块碎裂 模型太小，法线和贴图细节超过像素分辨率。 LOD 简化法线、降低细节贴图、远处减少色阶。

自研引擎模块拆分

不绑定图形 API 时，可以按下面的逻辑模块组织代码。这样后面无论接 Vulkan、D3D11 还是 D3D12，算法层都能保持一致。

模块 职责 核心数据
ToonMaterial 描述材质风格参数。 baseColor、shadowColor、ramp、threshold、softness、outlineWidth、rim。
ToonSurface 从 mesh 和贴图生成可渲染表面数据。 normal、uv、mask、materialId、surfaceFlags。
ToonLighting 主光、辅助光、色阶、高光、rim 的算法。 lightDir、lightColor、band、ramp、shadowVisibility。
ToonShadow 管理投影如何影响卡通暗面。 shadowStrength、shadowSoftness、receiverMask。
ToonOutline 生成外扩描边、屏幕边缘和结构线。 outlineNormal、lineWidth、lineColor、depthNormalEdge。
ToonCharacterRules 脸、头发、眼睛、特殊角色规则。 faceSDF、hairTangent、eyeHighlight、characterDirection。
ToonPostProcess 统一色调、抗锯齿、bloom、雾。 LUT、exposure、bloomMask、fogColor。

建议的渲染数据结构

struct ToonMaterial
{
    Color baseTint;
    Color shadowTint;
    Color specularColor;
    Color rimColor;
    Color outlineColor;

    float shadowThreshold;
    float shadowSoftness;
    float specThreshold;
    float specSoftness;
    float rimThreshold;
    float rimSoftness;
    float outlineWidth;

    int rampIndex;
    int materialClass; // Skin, Hair, Cloth, Metal, Eye, Effect
}

调试顺序

不要一开始就同时调光照、阴影、描边、后处理。卡通渲染最适合逐层打开。

  1. 只显示 BaseColor，确认贴图、UV、颜色空间正确。
  2. 显示 Normal，可视化模型法线是否平滑、脸部是否干净。
  3. 显示 N dot L 灰度图，确认主光方向和明暗变化。
  4. 打开单阈值 toon diffuse，只调 threshold 和 softness。
  5. 加入 shadowColor，确认暗部不是简单变黑。
  6. 加入 shadow map 或投影，只让投影影响明暗选择。
  7. 加入 specular band 和 rim light。
  8. 加入外扩描边，先只调外轮廓。
  9. 加入屏幕空间边缘，单独检查 depth edge 和 normal edge。
  10. 最后打开后处理和抗锯齿。

常见问题

现象 可能原因 修正方向
角色像塑料 高光太真实或太连续，暗部只是乘黑。 高光做成色块，暗部用指定 shadow color 或 ramp。
脸部很脏 直接使用真实法线和真实阴影。 编辑脸部法线，使用 face SDF 或脸部专用规则。
描边破碎 模型法线不连续、外扩方向错误、开口 mesh。 使用 outline normal，必要时给描边单独 mesh 或修模型。
画面太黑 把多个遮挡项连续相乘。 改为选择暗部色，不要无限乘黑。
色块边缘全是锯齿 硬 step 阈值，缺少过渡和抗锯齿。 增加 softness、使用边界抗锯齿、后处理 AA。
风格不统一 不同材质各自乱调，没有全局 ramp 和色彩规范。 建立共享 ramp、全局 light setup、统一 shadow tint 规则。

最实用的实现路线：先做“BaseColor + 主光二段色阶 + 暗部颜色 + 外扩描边”。这四个稳定后，再加 ramp、多段阴影、脸部 SDF、头发高光、屏幕空间边缘和后处理。
