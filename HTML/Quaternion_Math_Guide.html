0. 当前实现的表示约定

四元数不是“四个欧拉角”，而是一个实部加三个虚部。当前类型按 (x,y,z,w) 存储，其中 xyz 是虚部向量，w 是实部。本文世界坐标统一采用左手系：+X 向右、+Y 向上、+Z 向前。

存储顺序

Quaternion(x, y, z, w)

单位旋转

(0,0,0,1)

组合

lhs*rhs：先 rhs，后 lhs

向量旋转

q·(v,0)·q⁻¹

q = xi + yj + zk + w = (v, w)，其中 v=(x,y,z) 旋转四元数必须满足：||q|| = √(x²+y²+z²+w²) = 1
坐标系手性不会改写四元数代数 左手系不会改变 Hamilton 积、q·(v,0)·q⁻¹、共轭或逆的公式。手性主要决定哪个方向被解释为相机前方，以及 View/Projection 如何构造；不能因为改用左手 View/Projection 就随意取反四元数或旋转角。
Quaternion 的分量本身不是旋转轴和角度 只有通过轴角关系 xyz = axis·sin(θ/2)、w = cos(θ/2) 才能解释它们。

1. 为什么复数能旋转，四元数如何推广

1.1 二维复数旋转

二维单位复数可以写为欧拉公式：

z = cosθ + i sinθ，且 i²=-1

把二维点 p=x+iy 乘 z：

zp = (cosθ+i sinθ)(x+iy) = (x cosθ-y sinθ) + i(x sinθ+y cosθ)

实部和虚部恰好是二维旋转矩阵的结果。复数乘法把角度相加，因此连续组合稳定。

1.2 三维为什么需要三个虚数单位

三维旋转不再只绕一个固定平面，需要三个彼此正交的虚数方向 i、j、k：

i² = j² = k² = ijk = -1 ij = k，ji = -k jk = i，kj = -i ki = j，ik = -j

乘法次序交换后符号改变，所以四元数乘法不满足交换律。这与三维旋转的次序相关性完全一致。

1.3 为什么 q 与 -q 表示同一个旋转

(-q) · p · (-q)⁻¹ = (-q) · p · (-q⁻¹) = q · p · q⁻¹

负号出现两次后抵消。这叫四元数对 SO(3) 旋转群的双覆盖，也是插值时必须检查点积符号的原因。

2. 轴角公式为什么使用半角

给定单位旋转轴 n=(nₓ,nᵧ,n_z) 和旋转角 θ，单位四元数为：

q = ( n sin(θ/2), cos(θ/2) ) x = nₓ sin(θ/2) y = nᵧ sin(θ/2) z = n_z sin(θ/2) w = cos(θ/2)

本文沿用 Quaternion.hpp 的正角定义：axis=(0,0,1)、θ=+90° 时，+X 会旋转到 +Y。左手系的 +Z 同时是默认前向，但这不要求把轴角公式改成负半角。

2.1 单位长度验证

逐步验证 轴角四元数为什么天然是单位长度
前提 旋转轴 n 已归一化，因此 ||n||=1；令 α=θ/2，q=(n sinα, cosα)。
1
||q||²=||n sinα||² + cos²α
四元数长度平方等于虚部长度平方加实部平方。
2
||q||²=||n||²sin²α + cos²α
标量 sinα 可从向量长度中提出，平方后成为 sin²α。
3
||q||²=sin²α + cos²α = 1
代入 ||n||=1，再使用三角恒等式 sin²α+cos²α=1。
结论 ||q||=1，因此共轭 q* 同时也是逆 q⁻¹

2.2 半角来自左右各乘一次

向量旋转使用 qpq⁻¹，q 和 q⁻¹ 分别参与一次。展开后会出现双角公式：

2sin(θ/2)cos(θ/2) = sinθ cos²(θ/2)-sin²(θ/2) = cosθ

两次乘法把四元数中的半角合成为最终空间旋转角 θ。如果直接存 sinθ/cosθ，最终会产生 2θ。

3. Hamilton 积如何得到旋转组合

把四元数写成 q=(v,w)，v 是三维虚部。两个四元数相乘可整理为点积和叉积：

q₁q₂ = ( w₁v₂ + w₂v₁ + v₁×v₂, w₁w₂ - v₁·v₂ )

3.1 从基单位乘法展开

令 q₁=x₁i+y₁j+z₁k+w₁，q₂=x₂i+y₂j+z₂k+w₂。逐项相乘，使用 i²=j²=k²=-1 以及 ij=k 等规则，收集 i/j/k/实数项：

x = w₁x₂ + x₁w₂ + y₁z₂ - z₁y₂ y = w₁y₂ - x₁z₂ + y₁w₂ + z₁x₂ z = w₁z₂ + x₁y₂ - y₁x₂ + z₁w₂ w = w₁w₂ - x₁x₂ - y₁y₂ - z₁z₂

3.2 为什么乘法顺序就是旋转顺序

先用 q₂ 旋转 p，再用 q₁：

p' = q₂ p q₂⁻¹ p'' = q₁ p' q₁⁻¹ = q₁(q₂ p q₂⁻¹)q₁⁻¹ = (q₁q₂) p (q₂⁻¹q₁⁻¹) = (q₁q₂) p (q₁q₂)⁻¹

所以组合四元数 q1 * q2 表示先 q2、后 q1，与当前列向量矩阵组合规则一致。

不满足交换律 绕 X 90° 后绕 Y 90°，通常不等于绕 Y 后绕 X。代码中不能随意交换四元数乘法顺序。

4. 长度、共轭与逆四元数

长度
||q|| = √(x²+y²+z²+w²)
共轭
q* = (-x,-y,-z,w)
逆
q⁻¹ = q* / ||q||²
单位四元数
||q||=1，因此 q⁻¹=q*

4.1 为什么 q·q* 只剩实数

乘积推导 共轭为何能够构造四元数的逆
定义 q=(v,w)，q*=(-v,w)；Hamilton 积的虚部使用叉积，实部使用点积。
1
q q*=(v,w)(-v,w)
共轭只翻转虚部 v，实部 w 保持不变。
2
vector(q q*)=-wv + wv + v×(-v) = 0
前两项抵消；平行向量叉积为 0，所以虚部完全消失。
3
scalar(q q*)=w²-v·(-v)=w²+||v||²
v·v=||v||²，因此负向量把点积项的符号再次翻转。
4
q q*=(0, ||q||²)
因为 ||q||²=||v||²+w²，乘积只剩一个实标量。
5
q(q*||q||²)=1
当 q≠0 时同除以 ||q||²，便得到乘法单位元 (0,1)。
结论 q⁻¹ = q*||q||²；若 ||q||=1，则 q⁻¹=q*

因此 q(q*/||q||²)=1。对单位旋转，求逆只需翻转 xyz，代表绕相反方向旋转同样角度。

4.2 为什么要保持单位长度

Rotate 使用共轭等价于逆的前提是 ||q||=1。若长度漂移，旋转会混入缩放，矩阵转换也不再正交。

floatQuaternion rotation = Normalize(accumulatedRotation);
floatQuaternion inverse = Conjugate(rotation); // 单位旋转才成立

5. q·p·q⁻¹ 如何旋转向量

把三维向量 p 嵌入纯虚四元数 P=(p,0)。对单位 q=(u,s)：

P' = qPq⁻¹ q = (u,s)，q⁻¹=(-u,s)

5.1 第一次 Hamilton 积

qP = ( s p + u×p, -u·p )

5.2 再乘共轭并整理

完整展开 从 qPq⁻¹ 得到只含两次叉积的旋转公式
记号 q=(u,s) 为单位四元数，P=(p,0)，q⁻¹=(-u,s)；令 qP=(a,b)，其中 a=sp+u×p，b=-u·p。
1
P'=(a,b)(-u,s)
先保存第一次 Hamilton 积，再与共轭 q⁻¹ 相乘。
2
scalar(P')=bs+a·u=0
代入 a、b 后，-s(u·p)+s(p·u) 抵消，且 (u×p)·u=0；结果仍是纯虚四元数。
3
p'=b(-u)+sa+a×(-u)
按 Hamilton 积的虚部公式 w₁v₂+w₂v₁+v₁×v₂ 展开。
4
p'=(s²-||u||²)p + 2(u·p)u + 2s(u×p)
代入 a、b，并使用 p×u=-u×p 与 (u×p)×u=||u||²p-(u·p)u。
5
s²-||u||²=1-2||u||²
u×(u×p)=u(u·p)-||u||²p
单位条件 s²+||u||²=1；第二式是向量三重积公式。
6
p'=p+2s(u×p)+2u×(u×p)
把第 5 步两式代回第 4 步，得到与 Rodrigues 公式等价的形式。
实现形式 t=2(u×p)，p'=p+s·t+u×t

代码中 u 对应 q.xyz、s 对应 q.w。该形式只需两次叉积，无需真的创建 P、q⁻¹ 和两个临时四元数。

  1. 第一次叉积找出垂直于旋转轴和输入向量的方向。
  2. 乘 2 和实部 w，形成与 sinθ 对应的旋转分量。
  3. 第二次叉积形成与 1-cosθ 对应的轴向修正。
  4. 与原向量相加，得到 Rodrigues 旋转公式的同一结果。

5.3 90° Z 轴算例

u=(0,0,√2/2)，s=√2/2，p=(1,0,0) t = 2(u×p) = 2(0, √2/2, 0) = (0,√2,0) s·t = (0,1,0) u×t = (-1,0,0) p' = (1,0,0)+(0,1,0)+(-1,0,0) = (0,1,0)

6. 四元数与旋转矩阵互转

Quaternion、Matrix3x3、Matrix4x4 的双向转换、稳定分支与缩放/剪切处理，见 Quaternion 与 Matrix 转换详解。

6.1 从 qpq⁻¹ 收集 p 的系数

把上一节公式完全展开，并分别收集 pₓ、pᵧ、p_z 的系数，可得到：

R(q) = 1-2(y²+z²)2(xy-zw)2(xz+yw) 2(xy+zw)1-2(x²+z²)2(yz-xw) 2(xz-yw)2(yz+xw)1-2(x²+y²)

Matrix3x3FromQuaternion 先归一化 q，再缓存 xx、xy、wx 等乘积，避免重复计算。

6.2 为什么旋转矩阵是正交矩阵

单位四元数旋转保持长度和夹角，因此 R 的列向量仍是正交单位基：

RᵀR = I det(R) = 1 R⁻¹ = Rᵀ

6.3 从矩阵恢复四元数

矩阵迹 trace=R₀₀+R₁₁+R₂₂。由矩阵公式可得：

trace = 4w² - 1 → w = 1/2 √(trace+1)

当 trace>0 时用 w 作主分量稳定；当 trace≤0 时，选择最大对角元素对应的 x/y/z 分支，避免除以接近 0 的数。

  1. 计算 trace。
  2. 若 trace>0，先求 w，再由矩阵反对称部分求 xyz。
  3. 否则比较 R₀₀、R₁₁、R₂₂，选择最大的虚部分量先求。
  4. 最后 Normalize，消除输入矩阵的轻微数值误差。
输入必须接近纯旋转矩阵 带非均匀缩放或剪切的 3×3 不能直接转换为单位四元数。应先分解并正交化旋转部分。

7. Euler XYZ 如何组合

当前 QuaternionFromEulerXYZ 分别创建 X/Y/Z 轴旋转，然后返回：

q = normalize(q_z · q_y · q_x)

因为右侧最先作用于列向量，实际应用顺序是 X → Y → Z。

Euler 的优势

  • 编辑器中直观。
  • 单轴限制容易表达。
  • 适合序列化用户输入。

Euler 的问题

  • 旋转顺序影响结果。
  • 存在万向节锁。
  • 分量线性插值路径不自然。

7.1 万向节锁的本质

当中间旋转使两个旋转轴重合时，三个自由度退化为两个。四元数不会让物理姿态丢失自由度，但“从四元数转回某套 Euler 角”仍会遇到该参数化的奇异点。

8. Nlerp 与 Slerp 的推导

8.1 为什么普通 Lerp 不够

Lerp(q₀,q₁,t) = (1-t)q₀ + tq₁

线性插值沿四维空间的弦运动，会离开单位球，结果不再是单位四元数。

8.2 Nlerp

Nlerp(q₀,q₁,t) = normalize((1-t)q₀ + tq₁)

它便宜、连续、始终沿短弧附近移动，但角速度不完全恒定。短动画和逐帧姿态混合常用 Nlerp。

8.3 Slerp 权重如何得到

设单位四元数夹角为 Ω，点积给出余弦：

权重推导 从单位圆参数式推到 Slerp
前提 ||q₀||=||q₁||=1，0<Ω<π，t∈[0,1]；q₀、q₁ 位于 S³ 上同一大圆平面。
1
cosΩ=dot(q₀,q₁)
sinΩ=√(1-cos²Ω)
Ω=atan2(sinΩ,cosΩ)
单位向量点积给出夹角余弦；atan2 同时使用正弦和余弦，象限更明确。
2
e=q₁-q₀cosΩsinΩ
从 q₁ 中减去沿 q₀ 的投影并归一化，得到与 q₀ 垂直的单位切向 e。
3
q(t)=q₀cos(tΩ)+e sin(tΩ)
在正交基 {q₀,e} 中，这是单位圆上从 q₀ 转过 tΩ 的标准参数式。
4
q(t)=q₀cos(tΩ)+q₁-q₀cosΩsinΩsin(tΩ)
把第 2 步的 e 代入单位圆参数式。
5
q₀ 的系数=sinΩcos(tΩ)-cosΩsin(tΩ)sinΩ = sin((1-t)Ω)sinΩ
使用 sin(A-B)=sinA cosB-cosA sinB 合并 q₀ 的两项。
Slerp q(t)=q₀sin((1-t)Ω)sinΩ + q₁sin(tΩ)sinΩ

8.4 为什么点积为负要翻转终点

q₁ 与 -q₁ 是同一旋转，但在 S³ 上位于对径方向。若 dot<0，直接插值会走超过 180° 的长弧：

if dot(q₀,q₁) < 0: q₁ = -q₁ cosΩ = -cosΩ

8.5 小角度为什么退化为 Nlerp

Ω→0 时 sinΩ→0，Slerp 分母变小，容易损失精度；同时弦和弧几乎重合，因此用 Nlerp 更稳定。

0.35
当前优化为何不在 Slerp 末尾 Normalize 单位 q₀/q₁ 与球面权重理论上直接产生单位结果。与 DirectXMath 一样，当前实现依赖单位输入前置条件，省去一次 sqrt 和除法；小角度分支仍使用 Nlerp。

9. Quaternion 在 TRS 与层级中的位置

当前矩阵使用列向量，因此对象局部到世界的组合为：

M_world = T · R(q) · S p_world = T(R(S(p_local)))

9.1 父子层级

M_childWorld = M_parentWorld · M_childLocal q_childWorld = q_parentWorld · q_childLocal p_childWorld = p_parentWorld + Rotate(q_parentWorld, scaledLocalPosition)

如果父节点包含非均匀缩放，单独组合 position/rotation/scale 可能产生剪切，此时矩阵层级比分量级组合更可靠。

floatQuaternion rotation =
    QuaternionFromEulerXYZ(eulerRadians);
float4x4 localToWorld =
    TRSMatrix(position, rotation, scale);

float3 forward = math::Rotate(
    rotation,
    float3(0.0F, 0.0F, 1.0F));
float3 up = math::Rotate(
    rotation,
    float3(0.0F, 1.0F, 0.0F));

10. 数值稳定性与性能

10.1 什么时候归一化

场景建议原因
从轴角创建先 NormalizeSafe(axis)轴必须是单位向量
连续累乘旋转定期 Normalize(q)浮点误差会缓慢漂移
Rotate 热循环不要每次内部归一化调用方维护不变量更高效
Slerp 输入确保两端单位长度公式和免归一化优化的前提
Matrix→Quaternion函数末尾归一化吸收矩阵轻微误差

10.2 为什么当前 Rotate 比构造矩阵更便宜

旋转一个向量只需要两次叉积和若干乘加；先把 q 转 3×3 矩阵再乘向量，需要构造九个矩阵系数。若同一旋转作用于大量向量，预构造矩阵或批量 SIMD 才可能更划算。

10.3 常见错误

  • 把构造函数误写为 (w,x,y,z)。
  • 用角度直接传给要求弧度的 API。
  • 交换 parent * local 的顺序。
  • 对 q 的四个分量分别做 Euler 意义上的解释。
  • 忽略 q 与 -q 的双覆盖，导致插值绕远路。
  • 对非单位 q 直接使用共轭作为逆。
  • 误以为改成左手坐标系后，所有四元数角度或虚部都必须取反。
