1. 矩阵是什么

矩阵可以先理解为一个线性映射：它接收一个向量，输出另一个向量，并保持加法与数乘。

线性条件 T(u + v) = T(u) + T(v) T(kv) = kT(v)

1.1 每一列是基向量变换后的结果

设二维标准基为 e₁=(1,0)、e₂=(0,1)。如果一个变换把 e₁ 变为 (a,c)，把 e₂ 变为 (b,d)，矩阵就是：

M = [ T(e₁) T(e₂) ] = [ a b ] [ c d ]

任意向量 v = x e₁ + y e₂，根据线性条件：

T(v) = xT(e₁) + yT(e₂) = x(a,c) + y(b,d) = (ax + by, cx + dy)

这正是矩阵乘列向量。因此看一个变换矩阵时，前三列可以理解为变换后的 X/Y/Z 基轴，最后一列在齐次矩阵中保存平移。

1.2 为什么平移不是线性变换

平移 T(v)=v+t 不满足 T(0)=0，所以普通 3×3 线性矩阵不能表达三维平移。解决办法是增加齐次分量 w：

点： pₕ = (x, y, z, 1) 方向： vₕ = (x, y, z, 0) [ R t ] [ p ] = [ Rp + t ] [ 0 1 ] [ 1 ] [ 1 ] [ R t ] [ v ] = [ Rv ] [ 0 1 ] [ 0 ] [ 0 ]

同一个 4×4 矩阵因 w 不同自然区分点和方向，这就是 TransformPoint 与 TransformVector 的数学根源。

2. 矩阵乘法如何得到

2.1 矩阵乘向量：行与列做点积

输出第 i 个分量，是矩阵第 i 行与输入列向量的点积：

逐行推导 从线性映射得到输出分量
已知 M 是 R×C 矩阵，x 是 C 维列向量，y=Mx。
1
yᵢ=rowᵢ(M) · x
第 i 个输出只由 M 的第 i 行计算，它与 x 做点积。
2
yᵢ=Σj=0C-1 Mᵢⱼxⱼ
把点积展开：对应位置相乘，再把 C 项累加。
3
y₀=m₀₀x₀ + m₀₁x₁ + ... + m0,C-1xC-1
令 i=0，便得到输出向量第一个分量的实际计算式。
维度检查 (R×C)(C×1) = R×1
  1. 取输出行。要求第 0 个输出，就取矩阵第 0 行。
  2. 对应分量相乘。row[j] * vector[j]。
  3. 累加。得到该输出坐标。
  4. 对每一行重复。R×C 矩阵乘 C 维向量，得到 R 维向量。

数字算例

2103 × 34 = 2×3 + 1×40×3 + 3×4 = 1012

几何解释

也可以按列阅读：结果等于输入各分量对矩阵各列的线性组合，即 3·column0 + 4·column1。

2.2 矩阵乘矩阵：组合两个线性映射

设 C=A·B。为了让 C 作用于向量时等价于先 B 后 A：

组合推导 乘积矩阵的第 (i,j) 个元素
目标 令 C=A·B，使 C 作用于任意向量 v 时都等价于先执行 B、再执行 A。
1
Cv=A(Bv)
矩阵乘法必须保持两个线性映射的函数组合结果。
2
Cᵢⱼ=rowᵢ(A) · columnⱼ(B)
B 的第 j 列是基向量 eⱼ 经过 B 后的结果；再由 A 的第 i 行取出第 i 个分量。
3
Cᵢⱼ=Σₖ AᵢₖBₖⱼ
展开行列点积；k 同时遍历 A 的列和 B 的行。
结论 第 i 行 × 第 j 列 → C 的第 (i,j) 个元素
乘法不交换 通常 A·B ≠ B·A。例如“先绕原点旋转再平移”和“先平移再绕原点旋转”会得到不同位置。 
float4x4 world =
    TranslationMatrix(position) *
    Matrix4x4FromQuaternion(rotation) *
    ScaleMatrix(scale);

// 列向量从右向左执行：先 Scale，再 Rotation，最后 Translation。
float3 worldPoint = TransformPoint(world, localPoint);

2.3 为什么矩阵乘法满足结合律

矩阵代表函数组合，而函数组合满足结合律：

(A·B)·C·v = A·(B·(C·v)) = A·B·(C·v) 所以 (A·B)·C = A·(B·C)

结合律允许引擎预先计算 viewProjection = projection * view，随后每个对象只需再与 world 组合。

3. 行列式：缩放、方向与可逆性

行列式把一个方阵映射为标量。几何上，|det(M)| 是面积或体积的缩放倍数；符号表示定向是否翻转。

det = 2

面积/体积扩大 2 倍

det = -1

体积不变，但方向翻转

det = 0

空间被压扁，信息丢失，不可逆

3.1 二阶行列式为什么是 ad-bc

二维矩阵的两列 u=(a,c)、v=(b,d) 张成平行四边形，其有向面积为：

det [ a b ] = ad - bc [ c d ] 例：[ 2 1 ] → det = 2×3 - 1×0 = 6 [ 0 3 ]

3.2 三阶行列式

三维中 det([a b c]) = a·(b×c)，即一个向量与另外两个向量叉积的点积，得到平行六面体有向体积。

det(M) = m₀₀(m₁₁m₂₂ - m₁₂m₂₁) - m₀₁(m₁₀m₂₂ - m₁₂m₂₀) + m₀₂(m₁₀m₂₁ - m₁₁m₂₀)

3.3 余子式与 4×4 Laplace 展开

  1. 删除元素 mᵢⱼ 所在的第 i 行、第 j 列，得到子矩阵。
  2. 子矩阵的行列式叫 minor，记为 Mᵢⱼ。
  3. 乘棋盘符号 (-1)i+j 得到 cofactor Cᵢⱼ。
  4. 沿任意一行展开：det(A)=Σⱼ aᵢⱼCᵢⱼ。
第一行的符号： + - + - det(A) = a₀₀M₀₀ - a₀₁M₀₁ + a₀₂M₀₂ - a₀₃M₀₃

4. 逆矩阵：把变换撤销

若存在 A⁻¹ 使 A⁻¹A=AA⁻¹=I，则 A 可逆。它代表原变换的反函数。

4.1 二阶逆矩阵完整推导

完整推导 构造一个与 A 相乘后只剩单位矩阵的矩阵
已知 A=[[a,b],[c,d]]；要求 A⁻¹ 满足 AA⁻¹=A⁻¹A=I。
1
Δ=det(A)=ad-bc
Δ 是 A 的有向面积缩放倍数；Δ=0 表示空间被压扁，无法还原。
2
B = d-b-ca
交换主对角线 a、d，并让副对角线 b、c 变号，得到伴随矩阵 B。
3
AB = BA = ad-bc-ab+abcd-dcad-bc = ΔI
直接展开乘法，非对角项互相抵消，对角项都等于 Δ。
4
A(1ΔB)=1Δ(AB)=I
当且仅当 Δ≠0 时可以同除以 Δ，从 ΔI 恢复为 I。
结论 A⁻¹ = 1ad-bcd-b-ca
数字验证 A=[[2,1],[0,3]]
1
det(A)=2×3-1×0=6
行列式不为 0，因此逆矩阵存在。
2
A⁻¹ = 16 3-102 = 1/2-1/601/3
按公式交换、变号，再把每个元素除以 det(A)=6。
回代检查 A·A⁻¹ = 1001 = I

4.2 Gauss-Jordan 消元

通用 TryInverse<T,N> 把 A 与 I 并排，通过初等行变换把左边变成 I，右边就成为 A⁻¹。

[ A | I ] ──行变换──▶ [ I | A⁻¹ ]
  1. 当前列选择绝对值最大的主元，减少除以很小数造成的误差。
  2. 交换主元行到当前对角位置。
  3. 整行除以主元，让主元变成 1。
  4. 用当前行消掉同列其他元素。
  5. 每一列重复；若最大主元仍接近 0，则矩阵奇异。

4.3 伴随矩阵方法

A⁻¹ = adj(A) / det(A) adj(A) = cofactor(A)ᵀ

4×4 SIMD 快速路径不是逐行消元，而是并行计算余子式。当前实现把四行装入 SSE 寄存器，用三维叉积和点积构造伴随矩阵：

令四行拆为 (a,x), (b,y), (c,z), (d,w)，其中 a,b,c,d 为 float3： s = a × b t = c × d u = ay - bx v = cw - dz det = s·v + t·u r₀ = b × v + ty r₁ = v × a - tx r₂ = d × u + sw r₃ = u × c - sz 最后乘 1/det，并把四个“逆矩阵列”转置为行主序存储。
为何 SIMD 仍是同一个数学答案 SSE 只是在 128 位寄存器里同时计算四个 float。叉积、点积、余子式和 1/det 的公式没有改变；改变的是独立乘加的调度方式。

5. 仿射变换矩阵

5.1 平移、缩放与旋转

平移

T(t) = [ 1 0 0 tₓ ] [ 0 1 0 tᵧ ] [ 0 0 1 t_z ] [ 0 0 0 1 ]

缩放

S(s) = [ sₓ 0 0 0 ] [ 0 sᵧ 0 0 ] [ 0 0 s_z 0 ] [ 0 0 0 1 ]

绕 X 轴旋转时，X 坐标保持不变，YZ 平面按二维旋转：

Rₓ(θ) = [ 1 0 0 0 ] [ 0 cosθ -sinθ 0 ] [ 0 sinθ cosθ 0 ] [ 0 0 0 1 ]

5.2 Rodrigues 任意轴旋转

把向量 v 分解为沿单位轴 n 的平行分量和垂直分量。垂直分量在平面内旋转：

v∥ = n(n·v) v⊥ = v - v∥ R(v) = v cosθ + (n×v) sinθ + n(n·v)(1-cosθ)

把 v 的系数整理成矩阵，就得到 RotationAxisMatrix。

5.3 仿射逆为什么更简单

M = [ L t ] [ 0 1 ] M⁻¹ = [ L⁻¹ -L⁻¹t ] [ 0 1 ]

验证上式：

[ L t ][ L⁻¹ -L⁻¹t ] = [ I -t+t ] = [ I 0 ] [ 0 1 ][ 0 1 ] [ 0 1 ] [ 0 1 ]

因此标量回退会先检查最后一行是否为 (0,0,0,1)，若是，只求左上 3×3 的逆。

6. 观察矩阵如何构造

相机在世界里有位置和朝向。View 矩阵不是“把相机放进世界”，而是它的逆：把整个世界变换到相机局部空间。

6.1 右手 LookAt

  1. 前向。forward = normalize(target - eye)，相机观察方向。
  2. 右向。right = normalize(cross(forward, worldUp))。
  3. 真实上向。up = cross(right, forward)，保证三轴正交。
  4. 逆平移。每个轴与 -eye 点积，把相机位置移到原点。
ViewRH = [ rₓ rᵧ r_z -r·eye ] [ uₓ uᵧ u_z -u·eye ] [-fₓ -fᵧ -f_z f·eye ] [ 0 0 0 1 ]
worldUp 与 forward 平行 两者叉积接近零时无法确定 right。当前实现使用 NormalizeSafe 提供回退方向，但实际相机系统还应限制俯仰角或选择替代 up。

7. 投影矩阵的原理

7.1 透视来自相似三角形

右手相机看向 -Z。点 (x,y,z) 投到距离相机为 n 的近平面，根据相似三角形：

xₙ / n = x / (-z) → xₙ = n·x/(-z) yₙ / n = y / (-z) → yₙ = n·y/(-z)

除以 -z 不是线性的，不能直接写入普通矩阵。齐次坐标先让裁剪空间 w=-z，之后 GPU 做 perspective divide：

clip = P · viewPosition ndc = clip.xyz / clip.w clip.x = xScale·x clip.y = yScale·y clip.w = -z → ndc.x = xScale·x/(-z)

7.2 FOV 如何决定缩放

yScale = 1 / tan(fovY/2) xScale = yScale / aspect
60°

7.3 深度映射如何求系数

透视除法后深度形如 A + B/z。以 RH、ZO 深度 [0,1] 为例，要求 z=-n 映射 0，z=-f 映射 1：

联立求解 由近、远平面的目标深度确定 A 与 B
约定 右手相机看向 -Z；0<n<f；近平面 z=-n 映射 0，远平面 z=-f 映射 1。
1
ndc.z=Az+B-z
投影矩阵先生成 clip.z=Az+B、clip.w=-z，再由 GPU 做透视除法。
2
0=-An+Bn
B=An
代入近平面 z=-n；分母 n 非零，所以分子必须为 0。
3
1=-Af+Bf ⇒ -Af+B=f
代入远平面 z=-f，并把等式两边乘以 f。
4
-Af+An=f
A(n-f)=f
把 B=An 代入远平面方程，然后提取公因子 A。
5
A=fn-f
B=An=fnn-f
先解出 A，再回代近平面关系求 B。由于 n-f<0，两者均为负。
结果 clip.z = fn-fz + fnn-f，clip.w=-z

这说明深度不是线性分布，精度大量集中在近平面附近；near 设得过小会严重损失远处深度精度。

7.4 正交投影

正交投影没有除以 z，只把指定长方体线性映射到 NDC。以 x∈[l,r] 映射到 [-1,1] 为例：

x_ndc = ax + b al + b = -1 ar + b = 1 a = 2/(r-l) b = -(r+l)/(r-l)

y、z 同理解方程，因此正交矩阵适合 UI、编辑器网格、阴影方向光和无透视效果的工程视图。

8. 为什么法线要乘逆转置

切线 t 与法线 n 满足 n·t=0。顶点经过线性变换 M 后，切线变为 Mt。希望新法线 n' 仍垂直：

约束推导 让变换后的法线仍垂直于变换后的切线
前提 nᵀt=0，t'=Mt，并且 M 可逆；待求法线变换 n'=Nn。
1
n'ᵀt'=0
这是变换后必须继续满足的垂直约束。
2
n'ᵀMt=0
把已知的切线变换 t'=Mt 代入。
3
n'ᵀM=nᵀ
这样左式便还原成 nᵀt=0，对原切平面内任意 t 都成立。
4
n'ᵀ=nᵀM⁻¹
等式右侧同乘 M⁻¹，利用 MM⁻¹=I 消去 M。
5
n'=(M⁻¹)ᵀn
两边转置，并使用 (AB)ᵀ=BᵀAᵀ。
Normal Matrix N=(M⁻¹)ᵀ；计算后仍需归一化 n'

因此 NormalMatrix 提取 localToWorld 左上 3×3，求逆再转置。只有纯旋转或均匀缩放时，直接乘原矩阵后归一化才碰巧正确。

平移不影响法线 法线是方向，齐次 w=0；Normal Matrix 只需要 3×3 线性部分。若矩阵奇异，例如某轴缩放为 0，则法线逆矩阵不存在。

9. 当前 SIMD 乘法为什么快

普通通用模板通过动态索引循环访问 4×4 元素，MSVC 不一定展开。优化路径模仿 DirectXMath：一个矩阵行放进一个 128 位 SSE 寄存器。

9.1 矩阵乘矩阵

输出第 i 行 = lhsᵢ.x · rhsRow₀ + lhsᵢ.y · rhsRow₁ + lhsᵢ.z · rhsRow₂ + lhsᵢ.w · rhsRow₃
  1. 一次加载 RHS 四行，每行四个 float。
  2. 广播 LHS 当前行的 x/y/z/w。
  3. 四次并行乘法与三次并行加法得到完整输出行。
  4. 对 LHS 四行重复，RHS 寄存器可以复用。

9.2 为什么使用 loadu

float4 保持 16 字节大小但自然对齐仍可能是 4 字节。_mm_loadu_ps 不要求地址 16 字节对齐，因此不会改变顶点结构、序列化或常量数据布局。

9.3 常量求值和跨平台回退

constexpr float4x4 result = lhs * rhs;
// std::is_constant_evaluated() 为 true：走展开标量公式。

// CMake：关闭架构相关 SIMD，验证纯标量路径。
cmake -S . -B build_math_scalar -DMATH_ENABLE_SIMD=OFF
存储类型与计算类型分离 这是适合引擎的折中：外部仍使用紧凑 float3/float4/float4x4，热点进入函数后才加载到 SIMD 寄存器。

11. 推导与调试检查清单

公式检查

  • 先写清向量在左还是右。
  • 确认矩阵元素是 [row][column]。
  • 组合矩阵从实际执行顺序的反方向阅读。
  • 检查 RH/LH 与相机前向符号。
  • 检查 NDC 深度是 ZO 还是 NO。

数值检查

  • 用 M * Inverse(M) ≈ I 验证求逆。
  • 不要用精确相等比较一般浮点结果。
  • 近平面不要无限接近 0。
  • 非均匀缩放后的法线必须使用逆转置。
  • 奇异矩阵应返回失败，不应继续传播 NaN。

建议手算练习

  1. 手算二维矩阵 [[2,1],[0,3]] 的行列式、逆矩阵，并验证乘积。
  2. 构造“缩放 2 倍、绕 Z 旋转 90°、平移 (3,1)”的二维齐次矩阵。
  3. 从相似三角形重新推导 yScale=1/tan(fov/2)。
  4. 证明纯旋转矩阵满足 R⁻¹=Rᵀ。
  5. 用 n'=(M⁻¹)ᵀn 验证非均匀缩放后法线仍垂直切线。
