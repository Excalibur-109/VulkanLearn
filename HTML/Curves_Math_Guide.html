1. Curves.hpp 的完整能力图

参数曲线把参数 t 映射为标量、颜色、向量或空间位置：P(t)。当 P 是二维或三维向量时，它描述一条空间轨迹；当 P 是标量且范围为 0 到 1 时，它描述动画的时间节奏。

Bezier 插值

LinearBezier、QuadraticBezier、CubicBezier、通用 Bezier、BernsteinBasis。端点在曲线上，控制点决定切线与弯曲。

有理与切分 构造

RationalBezier、RationalQuadraticBezier、SplitQuadraticBezier、SplitCubicBezier。权重可精确表达圆锥曲线，切分不改变形状。

插值样条 经过点

CubicHermite、CardinalSpline、CatmullRom、CatmullRomNonUniform、KochanekBartels。适合路径、镜头、骨骼与关键帧。

B-Spline 逼近

CubicBSpline 一般不穿过控制点，但能通过局部四点影响得到更平滑的 C2 连续运动。

微分几何 测量

二次/三次一阶、二阶、三阶导数，Curvature 和 ApproximateCurveLength。用于朝向、曲率约束和匀速运动。

时间缓动 动画

EaseCurve 和 EvaluateEase 调度 31 条缓动函数。它们不是空间路径，而是把线性时间重新映射为非线性时间。

当前源码的关键区别：Bezier、Hermite、Cardinal、Catmull-Rom、TCB、B-Spline 都不钳制 t，允许主动外插。所有 Ease* 都会先 Saturate(t)，所以输入被限制在 [0, 1]。

实现内部的四个辅助函数

函数职责为什么需要它
detail::CurveLerp计算 start + (end-start)*t；对 2/3/4 维 Vector 有专门展开版本。让 De Casteljau、分割和 knot 插值共用同一种线性插值语义。
detail::EvaluateBezierLevel原地缩短控制点数组，每轮把相邻点 Lerp 为下一层。是通用 Bezier 的 De Casteljau 核心循环。
detail::CurveDistance标量使用 Abs(lhs-rhs)，Vector 使用 Distance(lhs,rhs)。让 ApproximateCurveLength 可同时处理数值曲线和空间向量曲线。
detail::KnotLerp按非均匀 knot 区间插值；区间接近零时返回起点。支撑 non-uniform Catmull-Rom，并避免重合控制点造成除零。

2. 参数曲线的基础：位置、速度、加速度

不要把 t 当作已走距离。它只是参数。二维轨迹 P(t)=(x(t),y(t)) 的一阶导数是速度或切线，二阶导数是加速度：

位置：P(t)；速度/切线：P'(t)=dP/dt；加速度：P''(t)=d²P/dt²；瞬时速度大小：|P'(t)|

t 不等于距离

相同 Δt 在弯曲段和直线段实际走过的距离不同。因此直接每帧递增 t，物体通常会视觉上忽快忽慢。匀速需要弧长查找表，详见第 6 节。

导数如何决定朝向

使用 Normalize(P'(t)) 作为前进方向。二维角度是 atan2(tangent.y, tangent.x)；三维则用切线与稳定的 up 向量构造旋转。

const float3 position = CubicBezier(p0, p1, p2, p3, t);
const float3 velocity = CubicBezierDerivative(p0, p1, p2, p3, t);
const float3 forward = Normalize(velocity); // 速度趋近 0 时应自行提供回退朝向。

3. Bezier：由 Lerp 推导出的曲线

3.1 LinearBezier

B(t) = (1-t)P0 + tP1 = P0 + t(P1-P0)，t ∈ [0,1]

t=0 得到 P0，t=1 得到 P1。导数 B'(t)=P1-P0 是常量，所以它是匀速直线。

3.2 QuadraticBezier

先在 P0-P1 与 P1-P2 上插值，再对两个中间点插值。将嵌套 Lerp 展开：

B(t) = (1-t)²P0 + 2(1-t)tP1 + t²P2

P1 通常不在曲线上，但控制端点切线：B'(0)=2(P1-P0)，B'(1)=2(P2-P1)。

3.3 CubicBezier

四个控制点的三次 Bezier 是图形软件、UI 和路径编辑中最常用的形式：

B(t) = (1-t)³P0 + 3(1-t)²tP1 + 3(1-t)t²P2 + t³P3

端点和把手

P0/P3 是端点。P1 决定从 P0 离开的方向，P2 决定进入 P3 的方向。B'(0)=3(P1-P0)，B'(1)=3(P3-P2)。

凸包性质

系数是非负且总和为 1 的 Bernstein 基函数，因此 t 在 [0,1] 时，曲线不会超出控制点的凸包。需要超调请使用外插或特殊 Ease。

4. 任意阶 Bezier、Bernstein 与精确切分

4.1 通用 Bezier：De Casteljau 算法

当前头文件的 Bezier(std::span<const Value>, t) 先复制控制点，再反复执行相邻 Lerp。每一轮活动点少一个，最后一个点就是答案。std::array 重载提供固定数量控制点的版本。

Qᵢ⁰=Pᵢ；Qᵢʳ=(1-t)Qᵢʳ⁻¹+tQᵢ₊₁ʳ⁻¹；B(t)=Q₀ⁿ

它比直接展开高阶多项式更稳定，并直接显示曲线的几何构造过程。内部 EvaluateBezierLevel 就是这一循环。

4.2 BernsteinBasis

Bᵢⁿ(t)=C(n,i)(1-t)ⁿ⁻ⁱtⁱ，C(n,i)=n!/[i!(n-i)!]

BernsteinBasis(degree,index,t) 返回这一项。源码先迭代计算组合数而不是直接算阶乘；当 index > degree 时返回 0。任意阶 Bezier 就是 ΣBᵢⁿ(t)Pᵢ。

4.3 SplitQuadraticBezier 与 SplitCubicBezier

De Casteljau 三角形的左边界和右边界分别就是切分后的控制点。三次曲线在 t 处切分：

左段=[P0,P01,P012,P0123]；右段=[P0123,P123,P23,P3]

P0123=B(t)，两段在该点连续连接，拼接后与原曲线严格相同。自适应细分、路径裁剪、碰撞近似和 tessellation 都可从这里出发。

5. 有理 Bezier：权重让圆弧变为精确

普通多项式 Bezier 无法精确表示圆。RationalBezier 为每个控制点加权，在齐次坐标中加权后再除以总权重：

R(t) = [Σ Bᵢⁿ(t)wᵢPᵢ] / [Σ Bᵢⁿ(t)wᵢ]

当所有 wᵢ 都为 1 时，它就是普通 Bezier。源码若权重个数与控制点个数不一致，或分母接近浮点 epsilon，会返回 Value{}，避免非法除法。

RationalQuadraticBezier：90 度单位圆弧示例

P0=(1,0)，P1=(1,1)，P2=(0,1)，w0=w2=1，w1=√2/2
  1. 在 t=0.5，三个未归一化基项是 1/4、√2/4、1/4。
  2. 分子 x=y=1/4+√2/4，分母=1/2+√2/4。
  3. 相除可得 R(0.5)=(√2/2,√2/2)，恰为单位圆的 45 度点。
权重的直觉：减小中间控制点的权重，曲线会远离它；增大权重则向它靠拢。只有权重选择符合圆锥曲线关系时才会得到精确圆弧。

6. 导数、曲率与弧长

6.1 当前实现中的 Bezier 导数

n 阶 Bezier 的一阶导数是 n-1 阶 Bezier，相邻差分控制点乘 n。因此不需要数值差分就能得到精确切线。

B'(t)=nΣBᵢⁿ⁻¹(t)(Pᵢ₊₁-Pᵢ)
函数公式作用
QuadraticBezierDerivative2(1-t)(P1-P0)+2t(P2-P1)速度、切线
QuadraticBezierSecondDerivative2(P2-2P1+P0)恒定加速度、曲率
CubicBezierDerivative3[(1-t)²(P1-P0)+2(1-t)t(P2-P1)+t²(P3-P2)]速度、切线
CubicBezierSecondDerivative6[(1-t)(P2-2P1+P0)+t(P3-2P2+P1)]弯曲变化
CubicBezierThirdDerivative6(P3-3P2+3P1-P0)三次曲线的常量 jerk

6.2 Curvature：曲线弯得有多急

二维：κ=|x'y''-y'x''|/|P'|³；三维：κ=|P'×P''|/|P'|³；曲率半径 R=1/κ

源码先计算 speedSquared=|P'|²。若它小于等于 epsilon，说明速度退化，函数返回 0；否则 speedSquared * sqrt(speedSquared) 就是分母 |P'|³。

二次 Bezier 的完整曲率计算

取 P0=(0,0)、P1=(1,2)、P2=(2,0)、t=0.5：

  1. P'(0.5)=2×0.5(P1-P0)+2×0.5(P2-P1)=(2,0)。
  2. P''=2(P2-2P1+P0)=(0,-8)。
  3. x'y''-y'x''=2×(-8)-0×0=-16，|P'|³=2³=8。
  4. κ=|-16|/8=2，因此 R=1/2=0.5。

蓝色是 t=0.5 的切线，黄色虚线圆是曲率圆，灰绿色折线是等参数采样得到的弧长近似。

6.3 ApproximateCurveLength

L=∫₀¹|P'(t)|dt ≈ Σᵢ₌₁ᴺ |P(i/N)-P((i-1)/N)|

源码按等参数间隔采样并累加相邻距离，subdivisions==0 时返回零。N 越大越准，但 t 等间隔不代表距离等间隔。

自适应精度

比较控制折线长度与端点弦长；误差过大就通过 SplitCubicBezier 继续二分。弯曲区自动得到更多采样，直线区不浪费计算。

匀速行进

预计算 (t, cumulativeLength) 表。按目标距离二分查表，再在两项 t 之间插值。对象按距离累加，不按 t 累加，即可近似匀速。

7. Hermite、Cardinal、Catmull-Rom、TCB 与 B-Spline

7.1 CubicHermite 与 CubicHermiteDerivative

Hermite 用端点 P0/P1 和端点切线 m0/m1 定义曲线。对三次多项式施加 P(0)=P0、P(1)=P1、P'(0)=m0、P'(1)=m1 四个条件，解得：

P(t)=h00P0+h10m0+h01P1+h11m1
h00=2t³-3t²+1，h10=t³-2t²+t，h01=-2t³+3t²，h11=t³-t²

CubicHermiteDerivative 对这四项求导。切线长度会改变曲线形状，不只是方向。

Hermite → Bezier：B0=P0，B1=P0+m0/3，B2=P1-m1/3，B3=P1
Bezier → Hermite：m0=3(B1-B0)，m1=3(B3-B2)

7.2 自动切线样条：所有函数的差异

函数核心计算端点适用场景
CardinalSplinem1=(1-T)(P2-P0)/2，m2=(1-T)(P3-P1)/2经过 P1/P2可调松紧的路径
CatmullRom调用 Cardinal，tension=0经过 P1/P2最常见自动切线插值
CatmullRomNonUniform按距离建立 knot，再进行三轮 KnotLerp经过 P1/P2点间距差异大时推荐
KochanekBartels由 Tension、Bias、Continuity 分别求 out/in 切线经过 P1/P2镜头和动画的艺术控制
CubicBSpline四个 B-Spline 基函数加权后除以 6一般不经过控制点需要 C2 连续平滑轨迹

7.3 非均匀 Catmull-Rom：CatmullRomNonUniform

kᵢ₊₁=kᵢ+|Pᵢ₊₁-Pᵢ|ᵅ；α=0 为 uniform，α=0.5 为 centripetal，α=1 为 chordal

源码将 alpha 钳制到 [0,1]。centripetal（0.5）通常能显著减少控制点间距很不均匀时的回环和自交。若相邻点重合，KnotLerp 检查到 knot 区间为零，直接返回起点而不是除零。

7.4 Kochanek-Bartels（TCB）

s=(1-T)/2
out=(P1-P0)s(1+B)(1+C)+(P2-P1)s(1-B)(1-C)
in=(P2-P1)s(1+B)(1-C)+(P3-P2)s(1-B)(1+C)

Tension 越大切线越短；Bias 使形状偏向前段或后段；Continuity 改变连接处的锐利程度。源码求出 out/in 后直接调用 Hermite。

上方对比 Hermite 与等价 Bezier、Cardinal tension、Non-uniform Catmull-Rom alpha、TCB 参数；下方绿色 Catmull-Rom 经过 P1/P2，蓝色 B-Spline 只逼近控制点。

8. EaseCurve：全部 31 条缓动曲线

缓动把线性时间 t 映射为 u=Ease(t)，再用 u 驱动 Lerp 或其他曲线：

value(t)=Lerp(start,end,Ease(t))
当前实现规则：所有 Ease 函数先 Saturate(t)。Exponential 在 t=0 和 t=1 时显式返回精确端点；Back、Elastic、Bounce 的函数值会按设计短暂越过 [0,1]。

沿路径匀速任意曲线 + 弧长表不能只线性推进 t。
  • 导数可能是零向量；调用 Normalize 前应检测长度，必要时沿用上一帧朝向。
  • 有理曲线的控制点与权重数量必须一致，总权重分母不能接近零。
  • 对渲染或碰撞的曲线离散化，应按屏幕误差或几何误差自适应细分，而不是一律使用固定大采样数。
  • Back、Elastic、Bounce 的超调是视觉设计；用于不允许越界的物理数值时，需要额外 Clamp。
