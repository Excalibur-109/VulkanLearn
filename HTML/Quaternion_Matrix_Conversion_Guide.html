0. 先固定约定，再谈公式

Quaternion 与 Matrix 的转换最容易出错的地方不是乘法，而是把不同资料的分量顺序、向量方向和矩阵布局混在一起。本文只使用当前 Math 库的实际约定。

Quaternion

(x,y,z,w)，xyz 为虚部，w 为实部。

Matrix 存储

行主序，matrix[row][column]。

向量约定

列向量右乘：p' = M * p。

世界坐标

左手系，+X 右、+Y 上、+Z 前。

旋转组合：A · B 表示先执行 B，再执行 A 四元数组合：q₁ · q₂ 表示先执行 q₂，再执行 q₁ 矩阵的列：R 的第 j 列 = 本地第 j 个基轴旋转后的坐标
本文最重要的判断 Quaternion 只能无损表示纯旋转。Matrix3x3 可以同时包含旋转、缩放、剪切和反射；Matrix4x4 还可包含平移与投影。因此从 Matrix 转 Quaternion 前，必须先证明或提取其中的纯旋转部分。

1. 转换前必须掌握的知识

1.1 点积、叉积与单位向量

a · b = aₓbₓ + aᵧbᵧ + a_zb_z = ||a|| ||b|| cosθ a × b = (aᵧb_z-a_zbᵧ, a_zbₓ-aₓb_z, aₓbᵧ-aᵧbₓ) normalize(a) = a / ||a||

点积判断长度和正交性；叉积产生垂直方向；归一化把长度变为 1。旋转矩阵的三个基轴必须两两点积为 0，且各自长度为 1。

1.2 旋转矩阵其实在保存三个旋转后的基轴

列向量约定下，矩阵乘单位基向量会直接取出对应列：

基向量为什么旋转矩阵的列就是局部坐标轴
定义eₓ=(1,0,0)ᵀ，eᵧ=(0,1,0)ᵀ，e_z=(0,0,1)ᵀ。
1
R eₓ = column₀(R)
eₓ 只让矩阵第 0 列的系数乘 1。
2
R eᵧ = column₁(R)
第 1 列是本地 +Y 旋转后在目标空间中的方向。
3
R e_z = column₂(R)
第 2 列是本地 +Z，也就是当前约定的前向轴。
行主序不等于基向量在行 行主序只说明连续内存先保存一整行；基轴在行还是列由数学乘法约定决定。当前库使用列向量，所以旋转后的基轴在矩阵列中。

1.3 正交旋转矩阵

纯旋转保持长度、夹角和空间朝向，因此矩阵必须满足：

RᵀR = I R⁻¹ = Rᵀ det(R) = +1 ||column₀|| = ||column₁|| = ||column₂|| = 1 columnᵢ · columnⱼ = 0，i ≠ j

det(R)=-1 虽然也可能满足 RᵀR=I，但它包含镜像反射，不是 Quaternion 能表示的 proper rotation。

1.4 齐次坐标为什么需要 4x4

3x3 能线性变换方向，却无法通过一次矩阵乘法表示平移。把三维点扩展为 (x,y,z,1)，把方向扩展为 (x,y,z,0) 后，4x4 才能统一旋转、缩放和平移：

[ R₃×₃ t ] [ p ] [ Rp+t ] [ 0 0 0 1 ] [ 1 ] = [ 1 ] [ R₃×₃ t ] [ d ] [ Rd ] [ 0 0 0 1 ] [ 0 ] = [ 0 ] ← 方向不受平移影响

1.5 单位 Quaternion

q = (u,w) = (x,y,z,w) ||q||² = x²+y²+z²+w² = 1 q⁻¹ = q* = (-x,-y,-z,w) 轴角：q = (n sin(θ/2), cos(θ/2))，||n||=1

单位条件使 q(v,0)q⁻¹ 只旋转、不缩放。q 与 -q 表示同一个三维旋转，因此矩阵转回的 Quaternion 不保证与原值逐分量相等。

1.6 叉积矩阵

把 u×p 写成矩阵乘法，可把四元数旋转快速展开成 3x3：

[u]× = 0-zyz0-x-yx0 ，使得 [u]×p = u×p

2. 三种表示之间能转换的前提

转换是否总能完成必要前提会丢失什么
Quaternion → Matrix3x3可以当前实现会先 Normalize；零长度回退 Identity无；但 q 与 -q 得到同一矩阵
Quaternion → Matrix4x4可以先生成 3x3，再嵌入齐次矩阵没有平移和缩放，默认只是纯旋转
Matrix3x3 → Quaternion有条件矩阵应接近正交，且 determinant 接近 +1矩阵中的缩放、剪切、反射无法保留
Matrix4x4 → Quaternion有条件先排除投影，提取左上 3x3 的纯旋转平移、缩放以及其他非旋转分量
Matrix3x3 → Matrix4x4可以新增齐次对角元素为 1，其余新增元素为 0无
Matrix4x4 → Matrix3x3可以提取取左上角 3x3平移、透视行列；提取结果可能仍含缩放/剪切
转换算法不能替代 Transform 分解 若 4x4 来自任意层级矩阵，正确问题通常是“如何将 affine matrix 分解为 T、R、S、shear”，而不是直接把左上 3x3 交给 QuaternionFromMatrix。

3. Quaternion 转 Matrix3x3 的完整推导

令单位 Quaternion q=(u,w)，其中 u=(x,y,z)。把向量 p 嵌入纯虚四元数 P=(p,0)，旋转定义为：

P' = q P q⁻¹ q = (u,w)，q⁻¹=(-u,w)

3.1 两次 Hamilton 积整理为向量公式

完整展开从 qPq⁻¹ 到矩阵形式
前提q 为单位 Quaternion；Hamilton 积写为 (v₁,s₁)(v₂,s₂)=(s₁v₂+s₂v₁+v₁×v₂, s₁s₂-v₁·v₂)。
1
qP = (wp + u×p, -u·p)
把 P=(p,0) 代入 Hamilton 积。
2
p'=(w²-||u||²)p + 2(u·p)u + 2w(u×p)
再乘 (-u,w)，使用向量三重积并收集 p 的项。
3
w²-||u||² = 1-2||u||²
单位条件 w²+||u||²=1。
4
p'=[(1-2u·u)I + 2uuᵀ + 2w[u]×]p
点积项写成外积矩阵，叉积项写成叉积矩阵。
结论R(q)=(1-2u·u)I + 2uuᵀ + 2w[u]×

3.2 展开三个矩阵项

(1-2u·u)I = diag(1-2x²-2y²-2z², 1-2x²-2y²-2z², 1-2x²-2y²-2z²) 2uuᵀ = 2 [ x² xy xz ] [ xy y² yz ] [ xz yz z² ] 2w[u]× = 2w [ 0 -z y ] [ z 0 -x ] [ -y x 0 ]

把对应位置相加。例如第一行：

R₀₀ = 1-2(x²+y²+z²)+2x² = 1-2(y²+z²) R₀₁ = 2xy-2wz = 2(xy-zw) R₀₂ = 2xz+2wy = 2(xz+yw)

3.3 最终 3x3 公式

R(q) = 1-2(y²+z²)2(xy-zw)2(xz+yw) 2(xy+zw)1-2(x²+z²)2(yz-xw) 2(xz-yw)2(yz+xw)1-2(x²+y²)

3.4 为什么这个矩阵必然是纯旋转

  • qPq⁻¹ 保持向量长度，因此 R 的每列长度为 1。
  • 旋转保持夹角，因此不同列仍互相正交。
  • q 从 Identity 连续变化时不会穿过 determinant 0，因此 det(R)=+1。
  • q 与 -q 的公式只有二次项和成对乘积，所有符号同时翻转后 R 不变。

4. Matrix3x3 与 Matrix4x4 如何转换

4.1 3x3 扩展为纯旋转 4x4

对齐次点 (p,1)，纯旋转不能改变 w，也没有平移。因此把 3x3 放在左上角，新加的最后一行和最后一列只有右下角是 1：

R₄ = R₀₀R₀₁R₀₂0 R₁₀R₁₁R₁₂0 R₂₀R₂₁R₂₂0 0001
R₄ · (pₓ,pᵧ,p_z,1)ᵀ = (R₃p,1)ᵀ R₄ · (dₓ,dᵧ,d_z,0)ᵀ = (R₃d,0)ᵀ

4.2 加入平移和缩放

当前列向量约定下，对象变换使用：

M = T · R₄(q) · S p_world = T(R(S(p_local)))

平移写入最后一列 M[0][3]、M[1][3]、M[2][3]。若只有 Quaternion 转 4x4，当前 Matrix4x4FromQuaternion 不会凭空加入平移或缩放。

4.3 4x4 提取 3x3

取左上角九个元素即可得到线性部分 A，但不要立刻把 A 称为旋转矩阵：

A = upperLeft3x3(M) 若 M=T·R·S，则 A=R·S 若 M 还含 shear，则 A=R·S·H 或其他分解顺序 若 M 是投影矩阵，则 A 根本不是对象旋转
“提取 3x3”与“提取旋转”不是同一件事 前者只是复制九个数；后者需要排除缩放、剪切和反射，确认结果满足正交旋转矩阵条件。

5. Matrix3x3 转 Quaternion 的完整推导

从上一节的九个矩阵元素反解 x、y、z、w。直接只使用 trace 的公式在 180° 附近会除以接近 0 的 w，因此实际代码必须选择数值最大的 Quaternion 分量作为主分量。

5.1 trace 为什么能求 w

迹推导由三个对角元素得到 w²
前提R 来自单位 q，因此 x²+y²+z²+w²=1。
1
trace = R₀₀+R₁₁+R₂₂
trace 是矩阵主对角线元素之和。
2
trace = 3-4(x²+y²+z²)
代入三个对角公式，每个虚部分量平方出现两次。
3
x²+y²+z² = 1-w²
来自单位 Quaternion 条件。
4
trace = 3-4(1-w²)=4w²-1
整理后得到只含 w² 的表达式。
结论w = 1/2 √(trace+1)，符号可由矩阵反对称部分配合确定

5.2 trace>0 分支

当前实现定义 scale = 2√(trace+1) = 4|w|。选择其中一个等价 Quaternion 符号后：

w 为主分量
S = 2√(trace+1)
w=S/4
x=(R₂₁-R₁₂)/S
y=(R₀₂-R₂₀)/S
z=(R₁₀-R₀₁)/S

差值来自矩阵的反对称部分：

R₂₁-R₁₂ = 2(yz+xw)-2(yz-xw) = 4xw R₀₂-R₂₀ = 2(xz+yw)-2(xz-yw) = 4yw R₁₀-R₀₁ = 2(xy+zw)-2(xy-zw) = 4zw

5.3 为什么 180° 附近不能继续用 w 分支

旋转角 θ 接近 180° 时 w=cos(θ/2) 接近 0，trace 接近 -1。此时 S 接近 0，再用差值除 S 会放大浮点误差。解决办法是找 x、y、z 中最大的那个先求。

5.4 最大对角元素的三个分支

类似地，由对角元素组合可得到：

1+R₀₀-R₁₁-R₂₂ = 4x² 1+R₁₁-R₀₀-R₂₂ = 4y² 1+R₂₂-R₀₀-R₁₁ = 4z²
x 为主分量
S=2√(1+R₀₀-R₁₁-R₂₂)
x=S/4
w=(R₂₁-R₁₂)/S
y=(R₀₁+R₁₀)/S
z=(R₀₂+R₂₀)/S
y 为主分量
S=2√(1+R₁₁-R₀₀-R₂₂)
y=S/4
w=(R₀₂-R₂₀)/S
x=(R₀₁+R₁₀)/S
z=(R₁₂+R₂₁)/S
z 为主分量
S=2√(1+R₂₂-R₀₀-R₁₁)
z=S/4
w=(R₁₀-R₀₁)/S
x=(R₀₂+R₂₀)/S
y=(R₁₂+R₂₁)/S

5.5 分支选择逻辑

if (trace > 0) {
    // w 分支
} else if (R00 > R11 && R00 > R22) {
    // x 分支
} else if (R11 > R22) {
    // y 分支
} else {
    // z 分支
}
q = Normalize(q);

“最大对角元素”与“最大虚部分量”对应，因为上面的三个对角组合分别等于 4x²、4y²、4z²。最后归一化用于吸收矩阵舍入误差，不是用来修复严重缩放或剪切。

输出符号不是唯一的 同一个矩阵可能恢复出 q 或 -q。若需要时间序列连续性，应与上一帧 Quaternion 做点积；若 dot<0，则把当前结果整体取负。

6. Matrix4x4 转 Quaternion 的正确流程

6.1 纯旋转 4x4

  1. 确认最后一行接近 (0,0,0,1)，它不是投影矩阵。
  2. 复制左上角 3x3 得到 R。
  3. 验证 RᵀR≈I、det(R)≈+1。
  4. 调用 QuaternionFromMatrix(R)。

6.2 含平移但不含缩放

M = R₀₀R₀₁R₀₂tₓ R₁₀R₁₁R₁₂tᵧ R₂₀R₂₁R₂₂t_z 0001

平移只在最后一列，完全不影响左上 3x3，因此可直接提取 R。Quaternion 只保存旋转，t 必须另外保存。

6.3 含正的非均匀缩放但没有剪切

若 M=T·R·S 且 S=diag(sx,sy,sz)，则左上 A 的三列分别是 R 的列乘对应 scale：

A.column₀ = sx · R.column₀ A.column₁ = sy · R.column₁ A.column₂ = sz · R.column₂ sx = ||A.column₀||，sy = ||A.column₁||，sz = ||A.column₂|| R.columnᵢ = A.columnᵢ / sᵢ

只有在没有 shear、scale 不接近 0 且符号问题已处理时，逐列归一化才能恢复 R。恢复后仍应检查列之间点积和 determinant。

float3x3 linear = ResizeMatrix<float, 3, 3>(matrix4x4);

float3 xAxis = linear.Column(0);
float3 yAxis = linear.Column(1);
float3 zAxis = linear.Column(2);

float3 scale(Length(xAxis), Length(yAxis), Length(zAxis));
// 先检查 scale 分量不接近 0，再归一化列。
xAxis /= scale.x;
yAxis /= scale.y;
zAxis /= scale.z;

float3x3 rotation{};
rotation.SetColumn(0, xAxis);
rotation.SetColumn(1, yAxis);
rotation.SetColumn(2, zAxis);

floatQuaternion q = QuaternionFromMatrix(rotation);
这段列归一化代码不是完整矩阵分解器 它只展示无 shear、正 scale 的简单情况。父子层级中的非均匀缩放与旋转组合可能产生 shear，此时必须使用更完整的 affine decomposition 或 polar decomposition。

7. 缩放、剪切和反射为什么麻烦

7.1 缩放破坏单位长度

纯旋转列长度为 1；缩放让列长度变成 scale。若把带 scale 的矩阵直接代入 trace 公式，trace=4w²-1 的推导前提已经不成立。

7.2 剪切破坏正交性

剪切会使基轴不再垂直。逐列 Normalize 只能修复长度，不能让两列重新正交：

normalize(c₀) · normalize(c₁) ≠ 0 → 仍然不是旋转矩阵 → QuaternionFromMatrix 得到的只是某个失真的近似结果

7.3 Gram-Schmidt 的近似正交化

对轻微数值漂移，可以按固定优先级正交化三个基轴：

x = normalize(c₀) y = normalize(c₁ - x·dot(x,c₁)) z = cross(x,y) y = cross(z,x) ← 再计算一次以减小误差

Gram-Schmidt 会偏向先处理的轴，不是距离原矩阵最近的旋转。适合清理轻微漂移，不适合一般 Transform 分解。

7.4 Polar decomposition

对可逆线性矩阵 A，可分为正交部分 R 与对称部分 S：

A = R S S = √(AᵀA) R = A S⁻¹ = A(AᵀA)⁻¹/²

R 是 Frobenius 范数意义下接近 A 的正交矩阵，S 吸收缩放和剪切。实现需要稳定的 3x3 对称矩阵平方根/逆平方根、奇异矩阵处理和 determinant 修正；成熟引擎通常使用经过测试的 decomposition 算法。

7.5 负缩放与反射

若 scale 有奇数个负轴，线性部分 determinant 为负。Quaternion 永远产生 det=+1，因此不能单独保存反射。分解器必须决定把负号放在哪个 scale 轴，同时让旋转部分恢复到 determinant +1。

输入情况列长度列点积det处理
纯旋转都约等于 1都约等于 0约 +1直接转换
正非均匀缩放等于各轴 scale仍约等于 0正但不一定 1列归一化后验证
剪切任意至少一对非 0任意完整分解或 polar decomposition
反射/奇数负 scale非负长度看不出符号可能仍为 0负保留负 scale，修正旋转 determinant
某轴 scale=0至少一列为 0信息丢失0旋转不可唯一恢复

8. 三个完整数值算例

8.1 绕 +Z 旋转 +90°

axis=(0,0,1)，θ=90° q=(0,0,sin45°,cos45°) =(0,0,√2/2,√2/2)

代入公式，x=0、y=0、z=w=√2/2：

R₃ = 0-10100001 R₄ = 0-100100000100001
R₃(1,0,0)ᵀ = (0,1,0)ᵀ → 当前正角约定下，+X 转到 +Y

8.2 从绕 +Y 60° 的矩阵恢复 Quaternion

R = 0.500.866025010-0.86602500.5
trace = 0.5+1+0.5 = 2 > 0 S = 2√(trace+1) = 2√3 ≈ 3.464102 w = S/4 ≈ 0.866025 x = (R₂₁-R₁₂)/S = 0 y = (R₀₂-R₂₀)/S = (0.866025-(-0.866025))/3.464102 = 0.5 z = (R₁₀-R₀₁)/S = 0 q=(0,0.5,0,0.866025)

8.3 绕 +X 180°，为何进入 x 分支

R = 1000-1000-1
trace = -1，w 分支的 S=0，不能使用 R₀₀ 最大，进入 x 分支 S = 2√(1+R₀₀-R₁₁-R₂₂) = 2√(1+1-(-1)-(-1)) = 4 x=S/4=1，y=z=w=0 q=(1,0,0,0)，也可能得到 -q=(-1,0,0,0)

10. 验证转换和排查错误

10.1 Quaternion 等价性

sameRotation(q₀,q₁) ⇔ |dot(normalize(q₀),normalize(q₁))| ≈ 1

绝对值同时接受 q 和 -q。也可以旋转三条单位基轴并比较结果，这比逐分量比较更符合真正目标。

10.2 Matrix 合法性

  • 检查三个列长度都接近 1。
  • 检查三对不同列的点积都接近 0。
  • 检查 Transpose(R)*R 接近 Identity。
  • 检查 determinant 接近 +1，而不是 -1 或 0。
  • 检查 R 与转换后的 q 对同一向量给出相同结果。

10.3 Round-trip 测试

floatQuaternion original = Normalize(input);
float3x3 matrix = Matrix3x3FromQuaternion(original);
floatQuaternion restored = QuaternionFromMatrix(matrix);

Check(
    Abs(Dot(original, restored)) > 1.0F - epsilon,
    "Quaternion matrix round-trip failed");

float3 probe = Normalize(float3(1.0F, 2.0F, 3.0F));
Check(Near(matrix * probe, Rotate(restored, probe), epsilon));

11. 公式与流程速查

Quaternion → Matrix3x3

先 normalize(q) R = [ 1-2(y²+z²) 2(xy-zw) 2(xz+yw) ] [ 2(xy+zw) 1-2(x²+z²) 2(yz-xw) ] [ 2(xz-yw) 2(yz+xw) 1-2(x²+y²) ]

Matrix3x3 → Quaternion

trace>0：以 w 为主分量 否则：比较 R₀₀、R₁₁、R₂₂，选择 x/y/z 主分量 最后 normalize(q) 切记：输入应满足 RᵀR≈I 且 det(R)≈+1

Matrix4x4 → Quaternion

1. 排除投影矩阵 2. 提取左上 3x3 线性部分 A 3. 若 A 含 scale/shear，先完成分解得到 R 4. 验证 RᵀR≈I、det(R)≈+1 5. QuaternionFromMatrix(R)

选择哪种表示

Quaternion

保存对象旋转、组合、插值；紧凑且没有 Euler 万向节锁。

Matrix3x3

批量变换方向和法线、保存线性部分；可包含非旋转信息。

Matrix4x4

统一对象到世界、相机与投影变换，适合渲染管线传递。
