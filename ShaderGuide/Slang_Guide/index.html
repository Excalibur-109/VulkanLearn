Slang Shader 写法指南

Slang 是一种 HLSL 风格的现代 shader 语言和编译器，目标是用一份源码生成多种后端输出，例如 Vulkan SPIR-V、D3D12 DXIL、D3D11 DXBC、Metal、WGSL、CUDA 或 CPU 代码。它适合你后面做多后端微型游戏引擎。

目录 Slang 是什么 本项目里的文件夹 推荐目录结构 常用写法 多后端编译 语义和资源绑定 代码整理规则 示例文件 检查清单

Slang 是什么

Slang 可以理解为“更适合大型工程和多后端的 HLSL”。它保留了大量 HLSL 语法习惯，例如 float3、cbuffer、Texture2D、SamplerState、SV_Position 语义，同时增加模块化、泛型、接口、反射 API、自动生成不同目标代码等能力。

语言主要输出适合场景
GLSLSPIR-V/OpenGL学习 Vulkan 和贴近 Vulkan 资源绑定。
HLSLDXBC/DXIL，也可编译到 SPIR-VD3D11/D3D12 或 Windows 图形开发。
SlangSPIR-V、DXIL、DXBC、MSL、WGSL、CUDA、CPU 等一份 shader 源码支持 Vulkan/D3D/Metal/WebGPU 等多后端。

多后端编译

Slang 的核心价值是同一份源码可以按不同 target 编译。下面命令需要你有 slangc.exe。

// Vulkan
slangc shaders/PbrLit.slang -entry VSMain -target spirv -profile vs_6_0 -o PbrLit.vert.spv
slangc shaders/PbrLit.slang -entry PSMain -target spirv -profile ps_6_0 -o PbrLit.frag.spv

// D3D12
slangc shaders/PbrLit.slang -entry VSMain -target dxil -profile vs_6_0 -o PbrLitVS.dxil
slangc shaders/PbrLit.slang -entry PSMain -target dxil -profile ps_6_0 -o PbrLitPS.dxil

// D3D11
slangc shaders/PbrLit.slang -entry VSMain -target dxbc -profile vs_5_0 -o PbrLitVS.dxbc
slangc shaders/PbrLit.slang -entry PSMain -target dxbc -profile ps_5_0 -o PbrLitPS.dxbc

语义和资源绑定

Slang 支持 HLSL 风格语义，所以你会继续看到 POSITION、NORMAL、TEXCOORD0、SV_Position、SV_Target0。编译到 Vulkan SPIR-V 时，编译器会把系统语义映射到 SPIR-V BuiltIn，把普通语义映射成 location。

Slang/HLSL 写法Vulkan 概念D3D 概念
SV_PositionBuiltIn Position系统值位置语义。
SV_Target0第 0 个 color attachment 输出 location第 0 个 render target。
TEXCOORD0普通 location 输入/输出普通插值语义。
register(b0, space0)set = 0, binding = 0CBV register b0 space0。
register(t1, space2)set = 2, binding = 1SRV register t1 space2。

代码整理规则

  • 通用数学、颜色空间、tone mapping 放 Common.slang。
  • CPU/GPU 共享布局放 SceneLayout.slang，字段按 16 字节对齐。
  • PBR、阴影、后处理函数拆成独立文件，入口函数只负责组装输入和输出。
  • 入口函数统一命名：VSMain、PSMain、CSMain，特殊 pass 用 ShadowVS、ShadowPS。
  • 多后端项目中优先使用 register(..., space...)，后续能自然映射 Vulkan set/binding 和 D3D register space。
  • 高级特性先少用。等基础 Vulkan/D3D 管线稳定后，再引入接口、泛型、自动微分和 reflection 生成绑定代码。

检查清单

  1. 每个 entry point 是否有正确的 [shader("...")] 属性。
  2. 语义是否和顶点布局、阶段输入输出一致。
  3. register 和 space 是否能映射到 Vulkan set/binding 和 D3D root signature。
  4. cbuffer、StructuredBuffer 的 C++ 结构体是否和 shader 布局一致。
  5. 编译到 Vulkan 时是否输出 SPIR-V，编译到 D3D12 时是否输出 DXIL。
  6. 是否使用 Slang reflection 读取资源绑定，避免 C++ 手写绑定表和 shader 漂移。
